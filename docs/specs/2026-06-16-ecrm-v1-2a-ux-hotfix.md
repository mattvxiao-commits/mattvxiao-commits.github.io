# ECRM V1.2a UI/UE 热修复说明

## 版本定位

V1.2a 是 V1.2 验收后的体验热修复版本，目标是修正已经进入核心操作流的空间浪费、状态操作低效和收款复核信息过载问题。

本版本不新增 V1.3/V1.4 的业务能力，不处理订单详情、库存流水、复杂仪表盘、云同步、复合二维码或部署教学。

## 问题背景

### 商品管理页面

当前商品管理页面在桌面横屏下仍占用过多顶部空间，页面标题、排序、新增按钮和统计信息分散展示，导致首屏可见商品数量不足。

商品列表中启用/停用状态也存在操作问题：

- 停用商品后，状态标签显示为停用，但操作按钮仍是停用。
- 停用按钮被禁用后，用户必须进入编辑弹窗才能重新启用商品。
- 点击停用没有二次确认，存在误操作风险。
- 状态标签位置在商品行内偏移，视觉结构不清晰。

### 售卖收款页面

进入收款确认后，顶部标题区仍偏展示型，占用过多纵向空间。

左侧“本单商品”复核区展示了完整商品编码。商品编码适合商品管理，不适合现场收款复核；在收款阶段继续展示编码会增加行高和视觉噪音，减少一屏可见商品数量。

## 修复范围

### P0：必须完成

1. 紧凑页头
   - 商品页顶部改为紧凑工具条。
   - 售卖页顶部改为紧凑工具条。
   - 标题、说明、刷新/新增、排序、统计信息尽量横向整合。
   - 保持移动端可换行，不允许文字互相遮挡。

2. 商品启用/停用双向切换
   - 启用商品显示“停用”操作。
   - 停用商品显示“启用”操作。
   - 点击停用弹窗确认。
   - 点击启用弹窗确认。
   - 操作处理中按钮临时禁用。
   - 启用/停用不改变商品其他字段。

3. 商品列表状态与操作布局修复
   - 状态标签固定在右侧操作区或标题区内，不再漂浮在商品行中部。
   - 操作按钮保持固定布局：编辑、复制、启用/停用。
   - 商品主信息区只展示商品名称、SPU、售价、编码、成本、库存、可售卖、可赠品。

4. 收款复核列表压缩
   - checkout 模式的“本单商品”列表不展示商品编码。
   - 商品行保留商品名、SPU、行类型、单价、数量、小计。
   - 商品行高度压缩，便于一屏显示更多商品。
   - 加购优惠行继续有浅色区分，但不额外增加明显行高。

### P1：尽量完成

1. 商品管理统计区压缩
   - 原大块统计条改为小型指标组。
   - 指标包含全部商品、启用中、可售、总库存。

2. 收款态顶部说明压缩
   - 保留必要说明，但降低字号和高度。
   - 收款态不要让页头挤压本单商品区域。

- 不新增商品高级搜索。
- 不新增多购物车暂存。

## 交互规则

### 停用确认

标题：确认停用商品？

正文：停用后该商品不会出现在售卖商品列表中，已有订单记录不受影响。

按钮：取消、确认停用

### 启用确认

标题：确认启用商品？

正文：启用后该商品会重新出现在符合条件的售卖商品列表中。

按钮：取消、确认启用

## 验收标准

### 商品管理

- 商品页顶部明显压缩，首屏能显示更多商品列表内容。
- 商品统计信息仍可见，但不再占用大块纵向空间。
- 启用商品可点击“停用”，并出现确认弹窗。
- 停用商品可点击“启用”，并出现确认弹窗。
- 停用后商品状态变为停用，按钮变为启用。
- 启用后商品状态变为启用，按钮变为停用。
- 状态标签位置稳定，不与商品主信息或按钮错位。
- 编辑和复制功能不回退。

### 售卖收款

- 售卖页顶部明显压缩。
- checkout 模式顶部明显压缩。
- 本单商品复核列表不显示商品编码。
- 本单商品复核列表仍显示商品名、SPU、单价、数量、小计和行类型。
- 本单商品行高较 V1.2 更紧凑。
- 收款方式、二维码、赠品选择、确认保存功能不回退。
