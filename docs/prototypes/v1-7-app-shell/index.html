Scheme D

混合响应式示意：桌面左侧导航 + 小屏底部导航

作为边界参考，不建议 V1.7a 直接全量实现，可作为长期产品化方向。

Desktop左侧导航 + 顶部状态栏 + 内容标题区
iPad 横屏左侧窄导航或顶部滚动导航二选一
iPad 竖屏顶部状态栏 + 底部主导航
Narrow顶部状态栏 + 底部主导航 + 页面内主操作

方案对比

A 当前结构进化版

最适合 V1.7a 快速落地，风险最低。

B 桌面管理工具版

长期扩展更清晰，但改动量更大。

C 现场操作优先版

iPad 体验最好，但要处理底部导航与购物车竞争。

D 混合响应式

长期方向参考，当前只做结构示意。
