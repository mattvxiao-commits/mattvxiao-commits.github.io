# ECRM V1.5 成本、毛利与订单 Excel 导出设计规格

## 1. 版本定位

V1.5 是 V1.4b-final 之后的经营分析增强版本，目标是在不引入云同步、不接入真实支付接口、不实现复杂拖拽仪表盘的前提下，补齐线下摆摊经营复盘中最关键的成本、毛利和表格化导出能力。

V1.5 分为三个开发子版本：

- V1.5a：订单成本快照。
- V1.5b：毛利、毛利率和成本相关仪表盘统计。
- V1.5c：订单记录 Excel 导出。

V1.5 完成后，建议执行总体验证、分支合并和正式版本 tag，建议 tag 为 `v1.5.0`。

## 2. 总体原则

### 2.1 成本快照优先

毛利统计必须基于订单下单时的成本快照，而不能直接读取商品当前成本价。

原因：

- 商品成本价后续可能调整。
- 如果历史订单毛利读取当前商品成本，会导致历史毛利被改写。
- 经营复盘需要可追溯的订单级成本数据。

因此 V1.5a 必须先完成订单成本快照，V1.5b 和 V1.5c 才能使用准确成本。

### 2.2 旧订单不伪造真实成本

V1.5a 之前已存在的订单明细没有成本快照。

处理原则：

- 不自动把当前商品成本价写入旧订单，避免制造看似真实但实际不可追溯的历史成本。
- 旧订单在毛利统计和 Excel 导出中标记为缺少成本快照。
- 准确毛利统计默认只纳入有成本快照的订单明细。

后续可以另行增加“按当前成本估算历史订单毛利”的辅助功能，但 V1.5 不做。

### 2.3 Excel 导出不是备份

V1.5c 的订单 Excel 导出用于统计、盘点和复盘，不用于恢复系统数据。

系统恢复仍使用 JSON 备份。

页面文案必须明确区分：

- JSON 备份：用于恢复系统数据。
- Excel 导出：用于 Excel 表格统计、盘点和复盘，不能用于恢复系统。

### 2.4 继续保持本地离线优先

V1.5 不引入：

- 云数据库。
- 账号系统。
- 多设备实时同步。
- 真实支付接口。
- 真实退款接口。
- 复杂拖拽图表。

## 3. V1.5a：订单成本快照

### 3.1 目标

在新订单下单时，把商品当时的成本价写入订单明细，形成可追溯成本快照。

V1.5a 完成后，新订单明细应包含：

- 成交信息。
- 商品名称、SPU、商品编码快照。
- 成本快照。
- 成本小计。
- 明细毛利。

### 3.2 数据结构

扩展 `OrderItem`：

```ts
export type OrderItem = {
  id: string;
  orderId: string;
  productId: string;
  productNameSnapshot: string;
  spuSnapshot: string;
  productCodeSnapshot?: string;
  quantity: number;
  originalUnitPrice: number;
  finalUnitPrice: number;
  lineType: OrderLineType;
  lineTotal: number;
  unitCostSnapshot?: number;
  costTotal?: number;
  grossProfit?: number;
};
```

字段说明：

- `unitCostSnapshot`：下单时商品单位成本价。
- `costTotal`：成本小计，等于 `unitCostSnapshot * quantity`。
- `grossProfit`：明细毛利，等于 `lineTotal - costTotal`。

字段使用可选值，是为了兼容 V1.5a 前的旧订单明细。

### 3.3 下单写入规则

在 `buildPaidOrder()` 生成 `OrderItem` 时，从对应 `Product.costPrice` 读取成本：

```text
unitCostSnapshot = product.costPrice
costTotal = product.costPrice * quantity
grossProfit = lineTotal - costTotal
```

不同明细类型规则：

- `normal`：正常计算成本和毛利。
- `discount_addon`：使用优惠后 `lineTotal` 计算毛利，成本仍按商品成本价计算。
- `gift`：`lineTotal` 为 0，成本仍按商品成本价计算，因此 `grossProfit` 为负数。

示例：

```text
普通商品：售价 20，成本 8，数量 2
lineTotal = 40
costTotal = 16
grossProfit = 24

优惠加购：成交价 3，成本 2，数量 3
lineTotal = 9
costTotal = 6
grossProfit = 3

赠品：成交价 0，成本 2，数量 5
lineTotal = 0
costTotal = 10
grossProfit = -10
```

### 3.4 数据库升级

Dexie 当前数据库版本为 2。

V1.5a 需要升级到版本 3。

由于新增字段不需要作为索引检索，`orderItems` store 可以继续使用原索引：

```ts
orderItems: "id, orderId, productId, lineType"
```

升级目的：

- 明确 schema 版本演进。
- 让 V1.5a 成本字段成为当前数据模型的一部分。
- 方便后续备份和测试区分 V1.5 前后数据。

### 3.5 备份升级

当前 JSON 备份版本为 3。

V1.5a 需要升级为备份版本 4。

备份版本 4 包含：

- 商品。
- 图片。
- 设置。
- 订单。
- 订单明细，包含成本快照字段。
- 库存流水。
- 人工退款记录。

导入兼容规则：

- Version 1：不含图片、不含退款、不含成本快照。
- Version 2：含图片、不含退款、不含成本快照。
- Version 3：含图片、含退款、不含成本快照。
- Version 4：含图片、含退款、含订单成本快照。

旧备份导入时：

- 不要求旧 `orderItems` 必须包含成本字段。
- 旧订单明细导入后成本字段保持缺失。
- 后续毛利统计按“缺少成本快照”处理。

### 3.6 订单详情展示

V1.5a 可以在订单详情明细行中增加简洁成本信息：

```text
成本 ¥X.XX
毛利 ¥Y.YY
```

对缺少成本快照的旧订单显示：

```text
缺少成本快照
```

展示原则：

- 不让成本信息压过现场售卖复核所需的商品、数量、价格和小计。
- 成本和毛利信息可以作为明细辅助信息显示。
- 不在 V1.5a 增加复杂毛利图表。

## 4. V1.5b：毛利与成本仪表盘

### 4.1 目标

在仪表盘增加固定毛利统计模块，让用户能在当前时间范围内查看：

- 成本。
- 毛利。
- 毛利率。
- SKU / SPU 毛利表现。
- 赠品成本和优惠成本压力。

### 4.2 统计范围

沿用 V1.4a 时间范围：

- 今日。
- 昨天。
- 近 3 天。
- 近 7 天。
- 自定义日期范围。

只统计当前范围内状态为 `paid` 的订单明细。

暂不统计：

- 作废订单。
- 待支付订单。
- 范围外订单。

### 4.3 准确毛利口径

准确毛利只统计有成本快照的订单明细。

字段判断：

```text
unitCostSnapshot、costTotal、grossProfit 均为有效数字
```

汇总口径：

```text
销售额 = 有成本快照明细的 lineTotal 合计
成本 = 有成本快照明细的 costTotal 合计
毛利 = 有成本快照明细的 grossProfit 合计
毛利率 = 毛利 / 销售额
赠品成本 = lineType 为 gift 的 costTotal 合计
```

当销售额为 0 时，毛利率显示为 0。

### 4.4 缺少成本快照提示

仪表盘需要显示当前范围缺少成本快照的订单或明细情况，避免用户误以为毛利覆盖全部订单。

建议增加：

```text
未纳入毛利明细数
未纳入毛利订单数
```

或在毛利模块说明中显示：

```text
当前范围有 X 条旧订单明细缺少成本快照，未纳入准确毛利。
```

### 4.5 建议新增指标

顶部或毛利模块中新增：

- 成本。
- 毛利。
- 毛利率。
- 赠品成本。

排行模块新增：

- SKU 毛利排行。
- SPU 毛利排行。
- 低毛利 SKU。

### 4.6 SKU 毛利排行

聚合维度：

- `productId`。

聚合字段：

- 商品名称快照。
- SPU 快照。
- 商品编码快照。
- 售出数量，不含 `gift`。
- 销售额。
- 成本。
- 毛利。
- 毛利率。

排序：

1. 毛利降序。
2. 销售额降序。
3. 商品名称中文排序。

最多展示 5 条。

### 4.7 SPU 毛利排行

聚合维度：

- `spuSnapshot`。

聚合字段：

- 售出数量，不含 `gift`。
- 销售额。
- 成本。
- 毛利。
- 毛利率。

排序：

1. 毛利降序。
2. 销售额降序。
3. SPU 中文排序。

最多展示 5 条。

### 4.8 低毛利 SKU

目的：

- 找出卖了但不太赚钱的商品。
- 帮助后续调整售价、优惠或赠品策略。

建议口径：

- 当前范围有销量。
- 有成本快照。
- 毛利率低于 20% 或毛利小于等于 0。

排序：

1. 毛利率升序。
2. 毛利升序。
3. 销售额降序。

最多展示 5 条。

### 4.9 退款边界

当前人工退款是订单级记录，不是商品级退款。

V1.5b 不把退款拆分到商品成本和 SKU 毛利。

页面或交付记录中需要说明：

- 毛利统计基于已支付订单明细成本快照。
- 当前人工退款金额仍在经营概览中体现。
- 商品级退款、退货入库和退款后毛利精确重算留给后续订单调整版本。

## 5. V1.5c：订单记录 Excel 导出

### 5.1 目标

新增订单记录 Excel 导出功能，导出 `.xlsx` 文件，用于：

- 整体表格化统计。
- 收摊后盘点。
- 活动复盘。
- Excel 透视表分析。
- 与外部账本或表格工具对照。

### 5.2 技术方案

建议新增 `xlsx` 依赖，用于生成真正的 Excel 工作簿。

原因：

- 用户明确需要 Excel 表格。
- 多 sheet 导出用 CSV 不合适。
- 手写 Office Open XML 成本较高，维护风险大。

导出仍在浏览器本地完成，不需要服务器。

### 5.3 页面入口

建议在设置页新增独立区块：

```text
表格导出
```

放在“备份与恢复”附近，但不放进同一个按钮组。

按钮：

```text
导出订单 Excel
```

辅助文案：

```text
Excel 用于统计、盘点和复盘，不能用于恢复系统数据。恢复数据请使用 JSON 备份。
```

### 5.4 导出文件命名

建议：

```text
ecrm-orders-YYYY-MM-DD.xlsx
```

如果后续支持按日期范围导出，可扩展为：

```text
ecrm-orders-YYYY-MM-DD-to-YYYY-MM-DD.xlsx
```

V1.5c 先导出全部本地订单记录，不增加复杂筛选。

### 5.5 Sheet 1：订单汇总

每行一个订单。

字段：

- 订单编号。
- 订单 ID。
- 订单状态。
- 支付方式。
- 创建时间。
- 支付时间。
- 作废时间。
- 商品原价合计。
- 优惠金额。
- 应收金额。
- 退款金额。
- 实收估算。
- 触发满赠档位。
- 是否赠品库存异常。
- 作废原因。
- 作废备注。

说明：

- 实收估算 = 应收金额 - 累计退款金额。
- 支付方式使用中文标签。
- 订单状态使用中文标签。

### 5.6 Sheet 2：订单明细

每行一个订单商品明细。

字段：

- 订单编号。
- 订单 ID。
- 订单状态。
- 支付时间。
- 商品 ID。
- 商品名称。
- SPU。
- 商品编码。
- 数量。
- 明细类型。
- 原单价。
- 成交单价。
- 销售小计。
- 单位成本快照。
- 成本小计。
- 毛利。
- 毛利率。
- 是否缺少成本快照。

明细类型中文：

- 普通。
- 优惠加购。
- 赠品。

缺少成本快照时：

- 单位成本快照为空。
- 成本小计为空。
- 毛利为空。
- 毛利率为空。
- 是否缺少成本快照 = 是。

### 5.7 Sheet 3：退款记录

每行一条人工退款记录。

字段：

- 订单编号。
- 订单 ID。
- 退款 ID。
- 退款时间。
- 退款金额。
- 退款方式。
- 退款原因。
- 备注。

### 5.8 Sheet 4：库存流水

每行一条库存流水。

字段：

- 时间。
- 商品 ID。
- 商品名称。
- 商品编码。
- 订单编号。
- 订单 ID。
- 变动数量。
- 变动原因。
- 变动前库存。
- 变动后库存。

说明：

- 商品名称和商品编码尽量从当前商品表补齐。
- 如果商品已删除或缺失，保留商品 ID，并在名称处标记为“商品不存在”。

### 5.9 Sheet 5：商品当前数据

每行一个当前商品。

字段：

- 商品 ID。
- 商品名称。
- SPU。
- SPU 编码。
- SKU 编码。
- 完整商品编码。
- 成本价。
- 售价。
- 当前库存。
- 是否可售。
- 是否赠品。
- 状态。
- 创建时间。
- 更新时间。

### 5.10 Sheet 6：导出说明

字段或内容：

- 导出时间。
- 系统版本。
- 文件用途说明。
- JSON 备份与 Excel 导出的区别。
- 成本快照说明。
- 旧订单缺少成本快照说明。
- 人工退款未按商品级拆分说明。
- 退款退货入库未实现说明。

## 7. 测试策略

### 7.1 V1.5a 测试

必须覆盖：

- 普通商品下单写入成本快照。
- 优惠加购下单写入成本快照。
- 赠品下单写入成本快照，毛利为负。
- 商品成本价后续变化不影响既有订单明细成本快照。
- 旧订单明细缺少成本字段时仍可读取和展示。
- 备份 version 4 导出包含成本字段。
- version 1、2、3 旧备份导入不要求成本字段。

### 7.2 V1.5b 测试

必须覆盖：

- 当前范围成本、毛利和毛利率计算。
- 赠品成本计入成本，降低毛利。
- 缺少成本快照的旧订单不计入准确毛利。
- 缺少成本快照数量提示。
- SKU 毛利排行。
- SPU 毛利排行。
- 低毛利 SKU。
- 时间范围切换后毛利统计随范围变化。

### 7.3 V1.5c 测试

必须覆盖：

- Excel 导出数据整理函数输出六个 sheet 的数据结构。
- 订单汇总字段中文表头。
- 订单明细字段中文表头。
- 缺少成本快照时导出为空值和“是”标记。
- 退款记录、库存流水、商品当前数据导出。
- 设置页点击导出按钮调用导出函数。
- 导出失败时显示脱敏中文错误。

## 8. 不做范围

V1.5 不做：

- 商品级退款。
- 退款退货入库。
- 换货记录。
- 补差价和退差价。
- 真实支付平台退款。
- 退款按商品成本精确拆分。
- 云同步。
- GitHub / 跨设备部署文档。
- 真实设备验收。
- 可配置拖拽仪表盘。

其中：

- GitHub / 跨设备部署文档在 V1.5-final 和 tag 后推进。
- 真实设备验收在 GitHub / 跨设备方案文档后推进。
- 商品级退款、换货、补差价和退款退货入库建议进入 V1.6。

V1.5-final 合并和 tag 前必须满足：

- V1.4b-final 已有总体验收记录。
- V1.5a、V1.5b、V1.5c 均有中文交付记录。

- 合并 `feature/ecrm-mvp` 到主分支。
- 创建 tag `v1.5.0`。
