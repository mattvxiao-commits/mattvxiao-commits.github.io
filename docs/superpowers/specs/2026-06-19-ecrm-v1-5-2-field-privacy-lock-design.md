# ECRM V1.5.2 现场售卖隐私锁设计规格

## 背景

线下摆摊收款时，操作者可能会把 Windows 笔记本、Mac 笔记本或 iPad 屏幕转向顾客，让顾客核对本单商品或扫描收款码。当前顶部导航仍可直接切换到商品、设置、仪表盘等页面，存在误触或被顾客主动点击后看到成本价、库存、经营统计、促销配置、备份导出等敏感信息的风险。

本版本新增“现场模式 / 售卖锁定模式”，目标不是提供银行级安全，而是为现场售卖提供明确、稳定的防误触和隐私遮挡能力。

## 目标

- 开启现场模式后，售卖流程保持可用。
- 开启现场模式后，商品、设置、仪表盘统一视为管理页面，进入前必须输入 PIN。
- 刷新页面、关闭后重新打开、PWA 重新启动后仍保持现场模式锁定状态。
- 操作者可以在现场快速重新锁定，避免临时解锁后转屏给顾客时暴露信息。
- PIN 不以明文保存。
- PIN 锁不随 JSON 备份迁移到其他设备，避免备份文件成为敏感凭据载体，也避免导入旧备份后把新设备锁死。

## 非目标

- 不做多用户账号、角色权限或登录系统。
- 不做云端认证、远程找回 PIN、短信/邮箱验证。
- 不做全量数据库加密。
- 不承诺抵御具备浏览器开发者工具能力的技术攻击者。
- 不在本版本实现“顾客全屏扫码展示模式”；该能力可作为后续 V1.5.3 或 V1.6a 独立优化。

## 功能范围

### 现场模式

现场模式启用后：

- `/sales` 售卖页保持可访问。
- `/products` 商品页进入前要求 PIN。
- `/settings` 设置页进入前要求 PIN。
- `/dashboard` 仪表盘进入前要求 PIN。
- 默认路由仍可跳转，但如果目标是受保护页面，需要先完成解锁。
- 顶部导航中的受保护页面仍可显示，但点击后应先弹出解锁弹窗，不应先展示页面内容。

### PIN 设置

- 第一次开启现场模式时要求设置 PIN。
- PIN 只能是 4 位或 6 位数字。
- 设置 PIN 时需要输入两次并一致。
- PIN 保存为哈希值，不保存明文。
- PIN 错误时提示“密码不正确，请重试。”
- 连续错误达到 5 次后，延迟 30 秒再允许继续尝试。

### 解锁行为

- 用户点击商品、设置、仪表盘时，如果现场模式已开启且当前未临时解锁，则弹出解锁弹窗。
- PIN 正确后进入用户原本点击的目标页面。
- PIN 正确后进入“临时解锁”状态，默认 5 分钟内可进入管理页面。
- 临时解锁期间，售卖页提供“重新锁定”入口，可立即结束临时解锁。
- 临时解锁到期后，再次进入管理页面需要重新输入 PIN。

### 设置入口

设置页新增“现场模式”配置区，但设置页本身在现场模式下受保护。

配置项包括：

- 开启现场模式。
- 设置或修改 PIN。
- 关闭现场模式。
- 显示当前状态：未开启 / 已锁定 / 临时解锁中。
- 提供“立即重新锁定”按钮。

### 售卖页入口

售卖页新增轻量入口：

- 未开启现场模式时，可显示“开启现场模式”入口，引导设置 PIN。
- 已开启且临时解锁时，显示“重新锁定”入口。
- 已开启且锁定时，可显示“现场模式已开启”状态。

售卖页不显示成本、利润等管理信息。

## 数据设计

建议在 `AppSettings` 中新增现场安全配置：

```ts
type FieldLockSettings = {
  enabled: boolean;
  pinHash?: string;
  pinSalt?: string;
  unlockExpiresAt?: string;
  failedAttempts: number;
  lockedUntil?: string;
};
```

默认值：

```ts
fieldLock: {
  enabled: false,
  failedAttempts: 0
}
```

哈希可使用浏览器 Web Crypto API 的 SHA-256，对 `pinSalt + pin` 进行摘要。该机制用于避免明文保存 PIN，不作为强加密安全边界。

## 备份规则

JSON 导出时：

- 不导出 `pinHash`。
- 不导出 `pinSalt`。
- 不导出 `unlockExpiresAt`。
- 不导出 `failedAttempts`。
- 不导出 `lockedUntil`。
- 可导出 `enabled: false`，或在导出数据中移除 `fieldLock` 后由导入端补默认值。

JSON 导入后：

- 现场模式默认关闭。
- 目标设备需要重新设置 PIN。
- 导入旧版本备份时自动补齐默认 `fieldLock` 配置。

## 路由与交互设计

新增管理页面访问保护层：

- 判断目标路由是否属于受保护路径。
- 如果现场模式未开启，允许访问。
- 如果现场模式已开启且临时解锁未过期，允许访问。
- 如果现场模式已开启且未解锁，阻止展示目标页面，显示 PIN 解锁弹窗。
- 用户取消解锁时停留在当前页面；如果是直接访问受保护 URL，则回到 `/sales`。

受保护路径：

```text
/products
/settings
/dashboard
```

非受保护路径：

```text
/sales
```

## 错误与边界

- 设置 PIN 两次不一致：提示“两次输入的密码不一致。”
- PIN 不是 4 位或 6 位数字：提示“请输入 4 位或 6 位数字密码。”
- 未设置 PIN 但尝试开启现场模式：阻止保存并提示先设置 PIN。
- 已开启现场模式但本地数据异常缺少 PIN 哈希：进入安全降级，管理页仍锁定；售卖页提示“现场模式配置异常，请在设置页重新配置。”设置页解锁可走重置流程。
- 忘记 PIN：提供“重置现场模式”入口，但必须二次确认；重置只关闭现场模式并清除 PIN，不删除商品、订单、库存数据。

## 测试范围

单元与组件测试至少覆盖：

- 默认现场模式关闭时，商品、设置、仪表盘可正常访问。
- 开启现场模式后，点击商品显示 PIN 解锁弹窗。
- PIN 错误不跳转，显示错误提示。
- 连续错误达到阈值后显示等待提示。
- PIN 正确后进入原目标页面。
- 临时解锁有效期内可访问管理页面。
- 立即重新锁定后再次访问管理页面要求 PIN。
- 刷新或重新挂载应用后现场模式仍开启。
- 售卖页在现场模式下可正常开单、加购、收款。
- JSON 导出不包含 PIN 哈希或盐。
- JSON 导入后现场模式默认关闭。

手动验收至少覆盖：

- Windows 浏览器。
- iPad Safari PWA。
- iPad 竖屏和横屏。
- 打开收款页后转屏给顾客时，误触商品/设置/仪表盘不会泄露内容。

## 建议版本归属

建议作为 `V1.5.2 现场售卖隐私锁` 独立小版本推进。

建议暂不和 README 修正、进销存增强、仪表盘增强混在同一次开发中，以便单独验证现场安全行为。
