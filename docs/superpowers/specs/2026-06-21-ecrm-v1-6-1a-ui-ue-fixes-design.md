# ECRM V1.6.1a 新功能 UI/UE 修复设计方案

## 1. 背景

V1.6.0 已支持混合订单、运营赠礼、人工赠送、其他非销售出库、仪表盘口径切换和 V1.6a 导出备份字段。实测截图显示，新功能虽然可用，但 UI/UE 存在明显问题：

- 购物车顶部新增的非销售出库入口占据过多空间，空购物车和有商品时都严重挤压商品列表。
- 仪表盘时间筛选、刷新、统计口径和自定义日期混在右侧，控件层级混乱，刷新按钮宽度异常。
- 运营赠礼配置只能指定 SKU，不能像满赠档位一样指定 SPU，无法覆盖“某个 SPU 内任选赠品 SKU”的现场场景。
- 全局顶部导航已经开始拥挤，但左侧导航属于更大范围框架改造，不纳入本轮。

## 2. 目标

V1.6.1a 的目标是修复 V1.6.0 新功能带来的高影响 UI/UE 问题，让用户能在真实摆摊场景中感受到明显改善。

本轮目标：

1. 购物车非销售入口紧凑化，不再占据购物车主体区域。
2. 购物车商品列表重新成为购物车主视觉和主操作区域。
3. 仪表盘筛选区重新分层，时间范围、刷新、统计口径、自定义日期各自有清晰位置。
4. 刷新按钮恢复正常按钮尺寸，和时间按钮视觉一致。
5. 运营赠礼支持“指定 SKU / 指定 SPU”两种默认目标配置。
6. 运营赠礼配置为 SPU 时，添加赠礼弹窗只显示该 SPU 下可作为赠品的 SKU，并由操作员手动选择具体 SKU。

## 3. 非目标

本轮不处理：

- 不把顶部横向导航改为左侧竖向导航。
- 不重做所有页面的全局布局。
- 不新增仪表盘图表或经营指标。
- 不扩展完整活动引擎。
- 不接入外部平台关注验证。
- 不改变人工赠送和其他出库的商品选择范围。

## 4. 购物车设计

### 4.1 当前问题

当前购物车把“运营赠礼 / 人工赠送 / 其他出库”放成大块按钮，空购物车时占据主要高度；有商品后仍挤压列表。该入口属于辅助开单动作，不应拥有高于商品明细的视觉优先级。

### 4.2 新设计

购物车顶部保留标题、件数、关闭按钮。

非销售入口改为一行紧凑操作条：

- `+ 运营赠礼`
- `+ 人工赠送`
- `+ 其他出库`

交互规则：

- 操作条高度控制在 36-40px。
- 空购物车时操作条仍显示，但主体区域显示“还没有选择商品”，不出现大面积空白按钮。
- 商品列表区域保持最大可用高度。
- 按钮使用次级按钮样式，但比普通按钮更紧凑。
- 运营赠礼未启用时隐藏 `+ 运营赠礼`，其他两个按钮仍保留。

## 5. 仪表盘筛选区设计

### 5.1 当前问题

仪表盘右侧控件混排，刷新按钮过宽，自定义日期输入区域占据过大空间，统计口径和时间周期在视觉上没有层级。

### 5.2 新设计

仪表盘顶部继续使用“左标题、右控件”结构，不做全局导航改造。

右侧控件分为三层：

1. 第一层：时间范围按钮 + 刷新按钮。
2. 第二层：统计口径按钮。
3. 第三层：仅自定义模式下显示日期范围输入。

布局规则：

- 第一层和第二层右对齐，按钮高度一致。
- 刷新按钮不跨整行，宽度只包裹图标和“刷新”文字。
- 自定义日期输入只在选择“自定义”后显示，宽度不超过筛选区主体宽度。
- 桌面端保持紧凑右侧控件；窄屏时允许换行，但不挤压标题文字。

## 6. 运营赠礼 SPU/SKU 配置

### 6.1 当前问题

运营赠礼只能配置默认 SKU。现场活动经常以“某个 SPU 内任选一个 SKU”为赠礼规则，例如某系列贴纸、小卡、吧唧。当前能力不够。

### 6.2 新数据结构

`CampaignGiftConfig` 增加目标类型字段：

```ts
targetType: "sku" | "spu";
defaultProductId: string;
defaultSpu: string;
```

兼容规则：

- 旧设置缺少 `targetType` 时默认视为 `sku`。
- 旧设置的 `defaultProductId` 保持不变。
- `defaultSpu` 缺失时默认空字符串。

### 6.3 设置页 UI

运营赠礼配置区改为：

- 启用运营赠礼。
- 运营活动名称。
- 默认赠礼目标类型：指定 SKU / 指定 SPU。
- 目标类型为 SKU 时显示“默认运营赠礼 SKU”下拉。
- 目标类型为 SPU 时显示“默认运营赠礼 SPU”下拉。

下拉范围：

- SKU 下拉只显示可作为赠品的 SKU。
- SPU 下拉只显示至少包含一个可作为赠品 SKU 的 SPU，并显示 SKU 数量。

### 6.4 售卖页添加逻辑

点击 `+ 运营赠礼`：

- 如果运营赠礼未启用，不显示入口。
- 如果配置为 SKU 且默认 SKU 有效，则可直接添加该 SKU。
- 如果配置为 SKU 但未配置默认 SKU，则打开运营赠礼选择弹窗，显示所有可作为赠品 SKU。
- 如果配置为 SPU 且默认 SPU 有效，则打开运营赠礼选择弹窗，只显示该 SPU 下可作为赠品的 SKU，由操作员手动选择具体 SKU。
- 如果配置为 SPU 但未配置默认 SPU，则打开运营赠礼选择弹窗，显示所有可作为赠品 SKU。

约束：

- 运营赠礼仍必须依附正常销售商品。
- 运营赠礼仍只能选择可作为赠品的 SKU。
- SPU 模式不自动挑选 SKU，避免库存和款式误判。

## 7. 测试策略

需要覆盖：

- CartPanel 非销售入口使用紧凑操作条，不再渲染大块按钮布局。
- DashboardPage 筛选区分为时间范围、统计口径、自定义日期三个区域，刷新按钮在时间范围区域内。
- SettingsPage 可以切换运营赠礼目标类型，并显示 SKU/SPU 对应下拉。
- 设置归一化函数能兼容旧 `campaignGift` 数据。
- SalesPage 在 SPU 模式点击 `+ 运营赠礼` 时打开限定 SPU 的赠品 SKU 选择弹窗。

## 8. 验收标准

- 空购物车时非销售入口不再占据大面积空白。
- 购物车有 2-4 件商品时，顶部非销售入口不挤压列表，商品卡片仍可快速扫描。
- 仪表盘顶部筛选区不再出现超宽刷新按钮。
- 自定义日期输入只在自定义模式下显示，并和统计口径按钮分层。
- 设置页运营赠礼可以选择指定 SKU 或指定 SPU。
- 指定 SPU 后，售卖页运营赠礼弹窗只出现该 SPU 下可作为赠品的 SKU。
- 原有 V1.6.0 混合订单、非销售出库、仪表盘口径、导出备份功能不回退。
