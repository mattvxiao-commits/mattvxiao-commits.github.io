# ECRM V1.6.4 仪表盘复盘结构整理设计方案

## 1. 背景

V1.6.3b 已补齐仪表盘经营复盘口径，包括正常销售、全部活动、运营赠礼、人工赠送、其他出库，以及订单性质、非销售拆分、经营成本口径等关键指标。当前数据口径已经更接近真实线下摆摊复盘需求，但页面结构仍是按历史迭代顺序自然堆叠。

随着模块数量增加，用户在收摊复盘时会遇到两个问题：

- 指标可用，但查找路径不够清晰。
- 销售、活动成本、毛利、库存、异常订单混在同一个长页面里，扫描成本偏高。

V1.6.4 的目标不是增加复杂 BI，也不是进入 V1.7 全局 UI 壳层改版，而是先把当前固定仪表盘整理成更符合经营复盘顺序的页面。

## 2. 目标

V1.6.4 聚焦仪表盘内部结构与复盘体验：

1. 按经营复盘顺序重新组织既有模块。
2. 增加仪表盘内部轻量分区导航，帮助快速跳转。
3. 使用分区标题明确每组指标的业务含义。
4. 保留所有 V1.6.3b 指标和统计口径，不改数据算法。
5. 保持工具型、高密度、低装饰的视觉风格。
6. 移动端保持可用，分区导航允许横向滚动或换行，不遮挡内容。

## 3. 非目标

V1.6.4 不处理：

- 不做 V1.7 全局导航 / app shell 大改版。
- 不改商品、售卖、设置页布局。
- 不新增拖拽式、飞书式、Excel 式复杂仪表盘。
- 不新增图表库。
- 不新增数据库 schema。
- 不调整订单、库存、退款、导出等业务逻辑。
- 不推送远端，不部署 GitHub Pages。

## 4. 页面结构

### 4.1 顶部筛选区保留

保留现有顶部结构：

- 标题：仪表盘。
- 当前统计范围。
- 日期范围：今日、昨天、近 3 天、近 7 天、自定义。
- 统计口径：正常销售、全部活动、运营赠礼、人工赠送、其他出库。
- 刷新按钮。

V1.6.4 不重构顶部筛选区，只允许轻微样式兼容。

### 4.2 新增分区导航

在错误提示 / 加载提示之后、数据内容之前，新增仪表盘内部导航：

- 概览。
- 销售。
- 活动。
- 毛利。
- 库存。
- 异常。

点击导航项跳转到同页对应分区。分区导航不改变路由，不影响 PWA 离线能力。

### 4.3 分区顺序

页面内容按以下顺序组织：

1. **经营概览**
   - 经营概览。
   - 售后概览。
   - 订单性质。

2. **销售表现**
   - 出库与客单。
   - 支付方式。
   - 热销 SKU。
   - 热销 SPU。
   - SPU 销售额。

3. **活动成本**
   - 活动效果。
   - 非销售拆分。
   - 经营成本口径。
   - 满赠触发。
   - 非销售出库分布。

4. **毛利分析**
   - 毛利概览。
   - SKU 毛利排行。
   - SPU 毛利排行。
   - 低毛利 SKU。

5. **库存风险**
   - 赠品消耗。
   - 低库存 SKU。
   - 售罄 SKU。
   - 高风险 SKU。
   - 滞销 SKU。
   - 补货建议。

6. **订单异常**
   - 异常订单。

## 5. UI/UE 规则

### 5.1 分区容器

新增 `dashboardGroup` 类用于分区：

- 分区本身不做大卡片。
- 分区标题为轻量标题行。
- 分区标题下直接放现有指标条或模块卡片。
- 分区之间使用合理垂直间距区分。

### 5.2 信息密度

继续保持高密度工具界面：

- 不新增大面积说明文案。
- 不新增插画、装饰背景或营销式区块。
- 分区标题文案短、直接。
- 复用现有 `dashboardMetricStrip`、`dashboardOperationsStrip`、`dashboardSection`。

### 5.3 响应式

桌面端：

- 页面仍为两列模块网格。
- 概览条跨整行。
- 分区内模块按现有两列网格排列。

移动端 / 窄屏：

- 分区内容单列。
- 分区导航可横向滚动或自然换行。
- 按钮和标题不重叠。

## 6. 技术设计

### 6.1 页面组件内拆分

在 `DashboardPage.tsx` 中只做局部整理：

- 新增分区配置数组。
- 新增 `DashboardGroup` 内部小组件。
- 将现有 JSX 模块按分区移动。

不新增独立文件，避免在 V1.7 app shell 前过早抽象。

### 6.2 测试

更新 `DashboardPage.test.tsx`：

- 断言分区导航存在且顺序正确。
- 断言主要分区 `经营概览 / 销售表现 / 活动成本 / 毛利分析 / 库存风险 / 订单异常` 可见。
- 断言关键模块仍可按 `aria-label` 找到。
- 断言日期范围和统计口径切换仍可用。

领域层 `dashboard.ts` 不修改，因此不新增领域层测试。

## 7. 验收标准

V1.6.4 完成后应满足：

- 仪表盘页面按 6 个复盘分区组织。
- 分区导航可见，点击可跳转到对应分区。
- 原有指标、排行、空态、异常提示仍可见。
- 日期范围切换、统计口径切换、刷新仍可用。
- 桌面端和窄屏布局不出现明显重叠。
- 仪表盘相关测试、全量测试、生产构建通过。
- 本轮只合并本地，不推送远端。
