# ECRM V1.6.3b 仪表盘经营复盘口径补齐设计方案

## 1. 背景

V1.6 已支持混合订单、运营赠礼、人工赠送、其他非销售出库，以及历史订单整单 / 明细级统计口径修正。V1.6.3a 又补齐了订单记录标签、修正弹窗和订单摘要刷新，解决了历史数据修正后订单预览仍显示旧性质的问题。

当前剩余风险集中在仪表盘复盘层：

- 正常销售、全部活动、运营赠礼、人工赠送、其他出库口径已经存在，但部分汇总指标仍不够贴近实际复盘。
- 用户需要在摆摊结束后快速判断：卖货本身赚了多少、运营活动花了多少、人工赠送和异常出库消耗了多少库存成本。
- 真实摆摊中存在“销售 + 运营赠礼”“纯人工赠送”“其他出库”“历史修正”等场景，仪表盘必须默认不污染正常销售，同时又能在全部活动口径下完整复盘。

V1.6.3b 不做飞书式复杂仪表盘，不做 V1.7 全局 UI 壳层改版，只补齐固定仪表盘里的经营复盘口径。

## 2. 目标

本轮目标是让仪表盘成为可靠的线下摆摊复盘页：

1. 默认“正常销售”口径只统计真实销售明细。
2. 纯非销售出库订单不计入正常销售订单数、销售额、客单价和销售排行。
3. 混合订单在正常销售口径下只按销售明细金额参与统计。
4. “全部活动”口径展示销售收入、销售成本、运营活动成本、非经营出库成本和全出库成本。
5. 新增或补齐订单数量拆分：正常销售订单、销售 + 赠送订单、纯非销售出库订单。
6. 新增或补齐非销售数量与成本拆分：满赠、运营赠礼、人工赠送、其他出库。
7. 新增或补齐活动后毛利率，便于判断销售收入扣除销售成本与运营活动成本后的真实经营结果。
8. 页面文案使用中文，强调口径含义，不使用冗余解释文案。

## 3. 非目标

V1.6.3b 不处理：

- 不重构全局导航、顶部栏、移动端 app shell。
- 不新增自定义图表、拖拽布局、复杂 BI 配置。
- 不新增商品级退款、换货、盘点、入库。
- 不改变订单、库存流水、退款的原始事实记录。
- 不改变购物车、收款页已验收布局。
- 不推送远端，不部署线上版本。

## 4. 统计口径原则

### 4.1 明细是仪表盘统计的主来源

订单金额、成本、排行和出库数量优先基于订单明细的归一化结果计算：

- `sale` 明细计入销售收入、销售成本、销售件数和销售排行。
- `non_sales / tier_gift` 计入满赠成本和非销售出库。
- `non_sales / campaign_gift` 计入运营赠礼成本和非销售出库。
- `non_sales / manual_gift` 计入人工赠送成本和非经营出库。
- `non_sales / other_non_sales` 计入其他出库成本和非经营出库。

### 4.2 正常销售口径

正常销售口径只统计销售明细。

要求：

- 销售额来自销售明细统计小计，不直接使用整单 `payableAmount`。
- 客单价分母只包含存在销售明细的已支付订单。
- 纯非销售出库订单不计入订单数和客单价分母。
- 退款仍按既有订单级退款逻辑处理；本轮不做商品级退款拆分。
- 销售排行、毛利排行、低毛利 SKU 不展示非销售明细。

### 4.3 全部活动口径

全部活动口径用于完整复盘现场库存与成本消耗。

要求：

- 销售收入仍只来自销售明细。
- 销售件数只来自销售明细。
- 非销售件数包含满赠、运营赠礼、人工赠送和其他出库。
- 运营活动成本 = 满赠成本 + 运营赠礼成本。
- 非经营出库成本 = 人工赠送成本 + 其他出库成本。
- 全出库成本 = 销售成本 + 运营活动成本 + 非经营出库成本。
- 活动后毛利 = 基础销售毛利 - 运营活动成本。
- 活动后毛利率 = 活动后毛利 / 销售收入。

### 4.4 单项非销售口径

运营赠礼、人工赠送、其他出库口径只展示对应原因的明细。

要求：

- 销售额显示为 0。
- 销售订单数显示为 0。
- 出库件数、成本、SKU 分布和原因分布可见。
- 不展示或不污染销售排行。

## 5. 领域模型新增 / 调整

### 5.1 `DashboardSummary`

保留既有字段，并调整含义：

- `paidAmount`：当前口径下统计销售额。正常销售和全部活动来自销售明细；非销售口径为 0。
- `paidOrderCount`：当前口径下有统计意义的订单数。正常销售口径只统计含销售明细订单；全部活动统计所有范围内已支付订单；单项非销售口径统计含对应非销售明细订单。

### 5.2 新增 `DashboardOrderNatureSummary`

新增订单性质拆分：

```ts
type DashboardOrderNatureSummary = {
  saleOrderCount: number;
  mixedOrderCount: number;
  nonSalesOrderCount: number;
  campaignGiftOrderCount: number;
  manualGiftOrderCount: number;
  otherNonSalesOrderCount: number;
};
```

用途：

- 页面可显示正常销售订单、销售 + 赠送、纯非销售出库。
- 辅助识别现场是否存在大量赠送 / 出库记录。

### 5.3 新增 `DashboardNonSalesBreakdown`

新增非销售出库拆分：

```ts
type DashboardNonSalesBreakdown = {
  tierGiftQuantity: number;
  campaignGiftQuantity: number;
  manualGiftQuantity: number;
  otherNonSalesQuantity: number;
  tierGiftCost: number;
  campaignGiftCost: number;
  manualGiftCost: number;
  otherNonSalesCost: number;
};
```

用途：

- 直接看出满赠、运营赠礼、人工赠送、其他出库各消耗多少数量和成本。
- 与 `activityCostSummary` 保持同一成本来源，避免两个地方口径不一致。

### 5.4 调整 `DashboardActivityCostSummary`

保留既有字段，补齐：

- `basicGrossMargin`
- `activityAdjustedGrossMargin`

毛利率规则：

- 销售收入为 0 时显示 0%。
- 四舍五入到两位小数，页面显示时去掉无意义的 `.0`。

## 6. 页面呈现

### 6.1 顶部筛选保持现状

V1.6.3b 不做顶部筛选大改版，保留：

- 日期范围：今日、昨天、近 3 天、近 7 天、自定义。
- 统计口径：正常销售、全部活动、运营赠礼、人工赠送、其他出库。
- 刷新按钮。

### 6.2 概览区调整

经营概览继续显示：

- 销售额。
- 退款。
- 实收。
- 订单。

其中“订单”遵循当前统计口径，不再无条件显示全部已支付订单数。

新增订单性质概览：

- 正常销售。
- 销售 + 赠送。
- 非销售出库。
- 运营赠礼订单。

新增非销售出库拆分：

- 满赠：数量 / 成本。
- 运营赠礼：数量 / 成本。
- 人工赠送：数量 / 成本。
- 其他出库：数量 / 成本。

### 6.3 经营成本口径

经营成本区显示：

- 优惠让利。
- 销售成本。
- 基础毛利。
- 基础毛利率。
- 运营活动成本。
- 活动后毛利。
- 活动后毛利率。
- 非经营出库。
- 全出库成本。

页面文案保持短句，不在卡片内写长说明。

## 7. 测试策略

### 7.1 领域层测试

新增或调整测试覆盖：

- 正常销售口径排除纯非销售订单的订单数和客单价。
- 混合订单在正常销售口径下只按销售明细金额计算。
- 全部活动口径正确拆分销售成本、满赠成本、运营赠礼成本、人工赠送成本、其他出库成本。
- 单项非销售口径的订单数、出库数量、成本和分布只包含目标原因。
- 活动后毛利率计算正确，销售收入为 0 时为 0。

### 7.2 页面测试

新增或调整测试覆盖：

- 页面显示订单性质概览。
- 页面显示非销售出库拆分。
- 页面显示基础毛利率和活动后毛利率。
- 切换“全部活动”和单项非销售口径后，指标随口径变化。

## 8. 验收标准

V1.6.3b 完成后应满足：

- 默认正常销售口径不被运营赠礼、人工赠送、其他出库污染。
- 纯非销售订单不再拉低正常销售客单价。
- 全部活动口径可以一次性看到销售、运营活动、非经营出库、全出库成本。
- 能直接区分满赠成本和运营赠礼成本。
- 能直接区分人工赠送成本和其他出库成本。
- 页面仍为轻量固定仪表盘，不出现复杂配置或大改版。
- 仪表盘相关测试、全量测试、生产构建通过。
- 本轮只合并本地，不推送远端。
