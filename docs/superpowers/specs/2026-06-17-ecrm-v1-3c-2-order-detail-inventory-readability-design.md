# ECRM V1.3c-2 订单详情库存流水可读性优化方案

## 1. 背景

V1.3b/V1.3c-1 已经实现订单作废、库存回滚、作废原因和备注记录。当前订单详情页的“库存流水”会把支付扣减、赠品扣减、作废回滚逐条完整展示，信息准确，但在订单作废后现场复核时会显得复杂。

V1.3c-2 的目标是降低作废订单详情的阅读成本，让操作员先看到“这笔订单库存发生了什么”，再按需展开完整流水。

## 2. 本版本目标

- 订单详情库存区域改为“库存摘要”优先展示。
- 支付扣减、赠品扣减、作废回滚分别汇总为易读指标。
- 完整库存流水保留，但放入可展开区域。
- 作废订单默认折叠完整流水，避免主视图过载。
- 未作废订单仍默认展开完整流水，方便查看正常扣减明细。
- 不改变订单、库存、备份、Dexie schema 等数据结构。

## 3. 本版本不包含

- 不新增订单调整表。
- 不新增退款表。
- 不新增订单状态。
- 不修改库存流水写入规则。
- 不修改备份版本。
- 不修改 Dexie schema。
- 不做 V1.4 仪表盘。

## 4. 交互设计

### 4.1 库存区域标题

标题从“库存流水”调整为“库存摘要”，副文案调整为“先看汇总，完整流水可展开复核”。

### 4.2 摘要指标

库存摘要展示三类指标：

- 售卖扣减：统计 `order_paid` 流水涉及的商品数和扣减件数。
- 赠品扣减：统计 `gift_order_paid` 流水涉及的商品数和扣减件数。
- 作废回滚：统计 `order_cancelled_rollback` 流水涉及的商品数和回滚件数。

展示文案规则：

- 有数量时：`1 个SKU / 2 件`
- 没有数量时：`0 个SKU / 0 件`
- 作废回滚为 0 时仍显示，避免用户不确定是否漏回滚。

### 4.3 完整流水

完整流水使用原有明细行，但放入 `<details>` 折叠区域：

- paid 订单默认展开。
- cancelled 订单或存在作废回滚流水时默认折叠。
- 折叠入口文案：`完整库存流水（N 条）`。

### 4.4 空流水

如果没有库存流水，摘要指标显示 0，完整流水区域显示固定中文空态：`暂无库存流水。`

## 5. 技术设计

### 5.1 新增领域函数

新增 `src/domain/orderInventorySummary.ts`。

职责：

- 输入 `InventoryLog[]`。
- 输出售卖扣减、赠品扣减、作废回滚三类汇总。
- 只做纯计算，不依赖 React，不读写数据库。

建议类型：

```ts
export type InventorySummaryMetric = {
  productCount: number;
  quantity: number;
};

export type OrderInventorySummary = {
  paidDeduction: InventorySummaryMetric;
  giftDeduction: InventorySummaryMetric;
  rollback: InventorySummaryMetric;
};
```

数量规则：

- `order_paid` 和 `gift_order_paid` 通常是负数，统计时取绝对值。
- `order_cancelled_rollback` 通常是正数，统计时取绝对值。
- 商品数按 `productId` 去重。

### 5.2 组件接入

修改 `src/components/OrderDetailDialog.tsx`：

- 引入 `buildOrderInventorySummary`。
- 使用摘要结果渲染三张紧凑指标。
- 完整流水保留现有行结构。
- 用 `<details>` 控制完整流水展开/折叠。

### 5.3 样式

修改 `src/styles.css`：

- 新增摘要指标网格样式。
- 新增完整流水折叠区域样式。
- 保持当前订单详情页 8px 圆角和紧凑信息密度。

## 6. 测试策略

### 6.1 领域测试

新增 `src/domain/orderInventorySummary.test.ts`，覆盖：

- 正确汇总售卖扣减、赠品扣减、作废回滚。
- 多条同一商品流水时商品数去重，数量累加。
- 空流水返回 0。

### 6.2 组件测试

修改 `src/components/OrderDetailDialog.test.tsx`，覆盖：

- 订单详情显示库存摘要指标。
- 作废订单的完整流水默认折叠。
- paid 订单的完整流水默认展开。
- 空流水显示 `暂无库存流水。`。

## 7. 验收标准

- 作废订单详情中可以先看到售卖扣减、赠品扣减、作废回滚摘要。
- 作废订单默认不直接铺开所有库存流水。
- 点击完整库存流水后仍能看到原有每条流水明细。
- 正常已支付订单仍默认展示库存流水明细。
- 全量测试通过。
- 生产构建通过。
