# ECRM V1.6.3a 订单统计口径修正体验补强设计

## 背景

V1.6 已支持在同一订单内记录正常销售、运营赠礼、人工赠送和其他非销售出库，也支持在订单详情中对历史订单做整单或单行统计口径修正。实测确认核心功能可用，但历史订单修正和现场非销售出库仍有几处体验问题：

- 运营活动快照、人工赠送备注、其他出库备注都只能手填。
- 运营活动快照没有默认读取设置页当前运营赠礼活动名称。
- 人工赠送和其他出库备注实际必填，但 UI 未前置标注。
- 错误提示出现后会挤压弹窗底部操作按钮。
- 修正成功后只有页面全局状态提示，订单详情弹窗内没有明显反馈。
- 订单记录总览无法快速识别订单性质和是否做过统计修正。
- 浏览器网页翻译可能把金额显示成双份文本，需要低成本防护。

## 目标

V1.6.3a 只收口订单统计口径修正和非销售出库录入体验，让用户能更快、更清楚地修正历史订单，并降低现场手填备注的操作成本。

## 范围

本版包含：

- 订单详情统计修正弹窗增加快速选项。
- 运营赠礼修正默认带入设置页当前运营活动名称。
- 人工赠送、其他出库备注必填提示前置。
- 非销售商品选择弹窗增加备注快速选项和固定错误提示区域。
- 订单详情内展示修正成功提示。
- 订单记录总览展示订单性质和已修正标签。
- 金额、订单号、商品编码等关键字段增加浏览器翻译防护。

本版不包含：

- 不改订单、退款、库存流水的业务写入规则。
- 不做仪表盘新增统计。
- 不做全局 App Shell UI 改版。
- 不做商品级退款、退货入库或订单重开。
- 不做可配置的备注模板管理页；快速选项先采用稳定内置模板。

## 交互设计

### 订单统计修正弹窗

弹窗仍复用整单和单行修正入口，保留四个修正选项：

- 销售
- 运营赠礼
- 人工赠送
- 其他出库

运营赠礼：

- “运营活动快照”默认填入设置页当前运营赠礼活动名称。
- 如果当前明细已有历史活动快照，则优先显示历史值，避免覆盖用户已记录内容。
- 提供快速选项：当前运营活动名称、关注社媒赠礼、现场互动赠礼、活动补赠。
- 非销售备注为可选，但提供快速选项：已完成关注、现场活动赠礼、补登记运营赠礼。

人工赠送：

- 非销售备注标注为“非销售备注（必填）”。
- 提供快速选项：好友赠送、摊主赠送、合作赠送、老客赠送。

其他出库：

- 非销售备注标注为“非销售备注（必填）”。
- 提供快速选项：样品出库、问题补偿、陈列损耗、盘点修正。

修正备注：

- 继续可选。
- 提供快速选项：历史订单补修正、现场复核修正、复盘口径修正。

错误提示：

- 弹窗内保留固定高度错误区域。
- 无错误时保留空间但不显示文本，避免按钮被顶动。

成功反馈：

- 修正成功后，订单详情弹窗内顶部显示成功提示。
- 文案为：统计口径已修正，原始支付、退款和库存流水未改动。

### 非销售商品选择弹窗

人工赠送和其他出库：

- 备注标题改为“备注（必填）”。
- placeholder 改为“例如：好友赠送 / 问题补偿 / 样品出库”。
- 提供与修正弹窗一致的备注快速选项。
- 错误提示区域固定高度，避免按钮下移。

运营赠礼：

- 不强制填写备注。
- 本版不改变运营赠礼商品选择逻辑。

### 订单记录总览

订单卡片右侧增加：

- 订单性质标签：正常销售、销售 + 赠送、非销售出库。
- 如果任一订单明细存在统计修正，则显示“已修正”标签。

由于订单列表接口当前只加载订单主表，不加载每笔订单明细，本版采用订单主表快照字段优先：

- `order.orderNature` 存在时使用快照值。
- 如果订单主表没有 `orderNature`，使用“正常销售”作为兼容默认值。
- 打开订单详情后，详情弹窗仍按实际明细实时推导订单性质。

本版不为了订单总览标签额外为每个订单加载明细，避免订单列表展开时产生大量 IndexedDB 查询。后续仪表盘口径版本如果需要更强的总览准确性，再统一评估订单主表冗余快照更新策略。

### 浏览器翻译防护

对以下内容增加 `translate="no"`，并补充 `notranslate` 类名：

- 金额。
- 订单号。
- 商品编码。
- 统计金额类字段。

该处理不能保证阻止所有浏览器插件，但能降低网页翻译把 `¥30.00` 追加为 `30.00 元` 的概率。

## 技术方案

- `OrderDetailDialog` 增加 `campaignGiftActivityName` prop。
- `SalesPage` 将 `campaignGift.activityName` 传入订单详情弹窗。
- `OrderDetailDialog` 内部增加快速选项渲染和点击填充逻辑。
- `SalesPage` 的 `NonSalesProductPicker` 增加快速备注选项。
- `orderHistory.ts` 增加订单总览展示 helper，避免页面直接散落推导逻辑。
- `styles.css` 增加快速选项、固定错误区域和订单性质标签样式。
- `index.html` 确认 `lang="zh-CN"`，必要时补齐。

  - 验证快速选项能填入活动快照、非销售备注和修正备注。
  - 验证必填备注标签和固定错误区域存在。

- `SalesPage.test.tsx`

  - 验证订单详情内修正成功提示可见。
  - 验证订单记录总览展示订单性质和已修正标签。

- `orderHistory.test.ts`
  - 验证订单总览标签 helper 对正常销售、混合、非销售和修正状态的输出。

## 验收路径

1. 打开售卖页订单记录，确认每条订单显示订单性质标签。
2. 打开已有订单详情，点击“修正统计口径”。
3. 选择“运营赠礼”，确认运营活动快照默认带入设置页活动名称，并可点快速选项修改。
4. 选择“人工赠送”或“其他出库”，确认备注显示必填并可通过快速选项填入。
5. 不填必填备注直接确认，确认错误提示出现但按钮区域不跳动。
6. 完成修正后，确认订单详情弹窗内出现成功提示。
7. 返回订单记录总览，确认修正过的订单显示“已修正”标签。
8. 开启浏览器网页翻译时，确认关键金额字段尽量不被翻译成双份金额。
