# ECRM V1.5.4 PWA 更新提示与缓存说明设计方案

## 背景

ECRM 已通过 GitHub Pages 提供 HTTPS 访问，并支持 PWA 离线使用。当前 `vite-plugin-pwa` 使用 `registerType: "autoUpdate"`，线上版本更新时，service worker 会在后台处理缓存更新，但用户缺少明确提示，容易出现“已经推送新版本，但当前页面仍看到旧版本”的困惑。

线下摆摊场景需要离线稳定性，但也需要在有网络时清楚知道版本是否已更新。`V1.5.4` 只处理 PWA 更新提示和缓存说明，不改业务数据、订单、库存、备份或现场模式。

## 目标

1. 当检测到 PWA 新版本可用时，在应用顶部显示轻量提示。
2. 用户点击提示按钮后，主动应用新版本并刷新页面。
3. 提供关闭提示入口，避免现场操作时遮挡。
4. README 和跨设备文档补充“看到旧版本时如何处理”的说明。
5. 保持离线优先能力，不破坏已缓存应用的离线使用。

## 非目标

- 不改变 IndexedDB 数据结构。
- 不清除用户业务数据。
- 不自动强制刷新页面。
- 不新增云同步。
- 不改变 GitHub Pages 部署方式。
- 不重做整体 UI。

## 方案

采用显式 PWA 更新提示：

- 将 PWA 注册方式从 `autoUpdate` 调整为 `prompt`。
- 在前端新增 `registerSW` 入口，保存 `updateServiceWorker` 回调。
- 新版本可用时，通过 React 状态显示顶部更新条。
- 更新条文案保持克制：

```text
发现新版本，建议在空闲时刷新更新。
```

- 主按钮：

```text
刷新更新
```

- 次按钮：

```text
稍后
```

点击“刷新更新”时调用 `updateServiceWorker(true)`，让 waiting service worker 接管并刷新页面。点击“稍后”只隐藏当前提示，不影响业务操作。

## UI 位置

更新条放在顶部导航下方、主内容上方：

- 不盖住商品列表、购物车或收款二维码。
- 使用紧凑横条样式。
- 桌面端横向排列，窄屏自动换行。
- 颜色使用当前应用的中性提示色，不使用危险或强警告样式。

## 数据安全

- 刷新更新只替换应用静态资源。
- 商品、订单、库存、收款码和设置仍保存在 IndexedDB。
- 不会因为点击更新提示而清空业务数据。
- 仍建议在摆摊前和收摊后导出 JSON 备份。

## 测试策略

自动化测试：

- `App` 在未检测到更新时不显示更新条。
- 触发新版本可用事件后显示更新条。
- 点击“稍后”隐藏更新条。
- 点击“刷新更新”调用注册模块保存的更新函数。

构建验证：

- `npm test`
- `npm run build`
- `git diff --check`

人工验收：

- 本地构建和预览页面正常打开。
- 更新条不影响顶部导航和主要页面布局。
- README 能解释线上更新后仍看到旧版本的处理方式。
