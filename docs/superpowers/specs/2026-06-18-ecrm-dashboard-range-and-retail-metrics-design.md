# ECRM 仪表盘时间范围与零售经营统计扩展设计方案

## 1. 背景

V1.4 已完成轻量固定仪表盘，可以查看今日销售、退款、实收、售后、热销 SKU、赠品消耗、低库存和异常订单。但当前仪表盘只支持“今日”口径，跨过 0 点后无法继续查看昨天或前几天的测试和摆摊数据。

线下摆摊场景中，收摊、盘点、补录退款、整理库存和核对收款经常发生在第二天凌晨。如果仪表盘只能查看当天，会直接影响复盘和对账。因此，下一阶段必须先建立时间范围切换能力，再继续增加零售经营、活动效果和库存风险统计。

用户已确认采用方案 A：

- V1.4a/V1.4b 先不做毛利。
- V1.4a 先做时间范围切换。
- V1.4b 增加经营统计、SPU 排行、支付方式、活动效果、库存风险和滞销统计。
- V1.5 再补订单明细成本快照，用可靠口径实现毛利、毛利率和库存价值。

## 2. 总体原则

### 2.1 时间范围优先

所有后续仪表盘统计都必须基于同一个时间范围口径。V1.4a 要先把“今日”扩展为可切换范围，避免后续每个模块重复实现日期判断。

### 2.2 固定实用面板

继续保持轻量固定仪表盘，不做飞书式拖拽组件，不做 Excel 式图表配置，不做复杂 BI。

### 2.3 口径明确

所有指标必须能解释清楚：

- 统计哪些订单。
- 按哪个时间字段归属。
- 作废订单是否计入。
- 退款是否影响实收。
- 赠品是否计入销量。
- 优惠让利如何计算。

### 2.4 不提前做不可靠毛利

毛利和库存价值是商业核心指标。当前订单明细没有 `costPriceSnapshot`，如果直接读取当前商品成本，会导致历史毛利随商品成本编辑而变化。因此 V1.4a/V1.4b 暂不做毛利，V1.5 先补成本快照，再做可靠毛利统计。

## 3. 版本拆分

### 3.1 V1.4a：时间范围切换

目标：解决跨天后无法查看昨天、近 3 天、近 7 天和自定义范围数据的问题。

功能：

- 增加时间范围控制：
  - 今日
  - 昨天
  - 近 3 天
  - 近 7 天
  - 自定义日期范围
- 所有 V1.4 已有模块改为按当前范围统计。
- 页面显示当前统计范围。
- 刷新按钮保留。
- 加载失败保护保留。

V1.4a 不新增经营指标，只改变统计范围能力。

### 3.2 V1.4b：零售经营统计扩展

目标：在时间范围底座上，增加对线下摆摊真正有用的经营、活动、支付和库存风险统计。

新增模块：

- 出库与客单：
  - 售出件数。
  - 赠品件数。
  - 总出库件数。
  - 客单价。
- SPU 表现：
  - 热销 SPU 排名。
  - SPU 销售额排名。
- 支付方式统计：
  - 微信、支付宝、现金、其他订单数。
  - 微信、支付宝、现金、其他收款金额。
- 活动效果：
  - 优惠加购件数。
  - 优惠让利金额。
  - 参与优惠订单数。
  - 满赠档位触发次数。
- 库存风险：
  - 售罄 SKU。
  - 低库存 SKU。
  - 高风险 SKU。
  - 滞销 SKU。
  - 补货建议。

V1.4b 不新增数据库 schema，优先基于已有 `orders`、`orderItems`、`orderRefunds` 和 `products` 计算。

### 3.3 V1.5：成本快照、毛利和库存价值

目标：先补可靠数据基础，再做商业核心指标。

数据结构建议：

- 给 `OrderItem` 增加 `costPriceSnapshot`。
- 新订单下单时写入当时商品成本。
- 备份导出/导入支持新字段。
- 旧订单缺少成本快照时，在毛利统计中标记为无法精确计算或按后续迁移策略处理。

新增指标：

- 范围销售额。
- 范围成本。
- 范围毛利。
- 毛利率。
- SKU 毛利排行。
- SPU 毛利排行。
- 当前库存成本总额。
- 当前库存按售价总额。
- 仅售卖 SKU 库存金额。
- 仅赠品库存金额。

## 4. V1.4a 时间范围设计

### 4.1 时间范围选项

页面顶部增加时间范围控制，建议使用分段按钮：

- 今日
- 昨天
- 近 3 天
- 近 7 天
- 自定义

选择“自定义”后显示：

- 开始日期。
- 结束日期。

### 4.2 时间范围口径

所有时间均按本地浏览器时间计算。

- 今日：当天 `00:00:00.000` 到当前时间。
- 昨天：昨天 `00:00:00.000` 到昨天 `23:59:59.999`。
- 近 3 天：包含今天在内的最近 3 个自然日，从起始日 `00:00:00.000` 到当前时间。
- 近 7 天：包含今天在内的最近 7 个自然日，从起始日 `00:00:00.000` 到当前时间。
- 自定义日期范围：开始日期 `00:00:00.000` 到结束日期 `23:59:59.999`。

如果自定义结束日期早于开始日期：

- 不执行统计。
- 显示中文错误：`结束日期不能早于开始日期。`

### 4.3 订单归属时间

已支付订单：

- 优先使用 `paidAt`。
- 缺失时使用 `createdAt`。

作废订单：

- 使用 `cancelledAt`。

退款记录：

- 使用 `OrderRefund.createdAt`。

异常订单：

- 已作废订单按 `cancelledAt` 归属范围。
- 已支付且退款/赠品异常订单按 `paidAt ?? createdAt` 归属范围。

### 4.4 文案规则

为避免近 3 天、近 7 天和自定义范围时文案过长，指标标签建议使用范围中性文案：

- 销售额。
- 退款。
- 实收。
- 订单。
- 作废订单。
- 部分退款。
- 已退款。
- 作废备注。

在标题下方显示当前范围，例如：

- `统计范围：2026-06-18 00:00 - 当前`
- `统计范围：2026-06-17 00:00 - 2026-06-17 23:59`
- `统计范围：2026-06-12 00:00 - 当前`

## 5. V1.4b 指标设计

### 5.1 出库与客单

新增指标：

- 售出件数：
  - 统计范围内已支付订单的 `normal` + `discount_addon` 明细数量。
- 赠品件数：
  - 统计范围内已支付订单的 `gift` 明细数量。
- 总出库件数：
  - 售出件数 + 赠品件数。
- 客单价：
  - 范围实收 / 范围已支付订单数。
  - 如果订单数为 0，则显示 `¥0.00`。

### 5.2 SPU 表现

新增排行：

- 热销 SPU：
  - 按范围内售出件数倒序。
  - 只统计 `normal` 和 `discount_addon`。
- SPU 销售额：
  - 按范围内销售金额倒序。
  - 只统计 `normal` 和 `discount_addon`。

每行显示：

- SPU。
- 售出件数。
- 销售额。

### 5.3 支付方式统计

统计范围内已支付订单：

- 微信订单数、微信收款金额。
- 支付宝订单数、支付宝收款金额。
- 现金订单数、现金收款金额。
- 其他订单数、其他收款金额。

作废订单不计入收款金额。

退款按 V1.4b 第一版暂不拆分到支付方式；后续可增加“按退款方式统计”模块。

### 5.4 活动效果

基于订单明细和订单促销字段计算：

- 优惠加购件数：
  - `lineType = discount_addon` 的数量合计。
- 优惠让利金额：
  - `discount_addon` 明细中 `(originalUnitPrice - finalUnitPrice) * quantity` 合计。
- 参与优惠订单数：
  - 至少包含一条 `discount_addon` 明细的订单数。
- 满赠档位触发次数：
  - 按 `Order.triggeredGiftTier` 聚合。
  - 只统计已支付订单。

### 5.5 库存风险

库存风险使用当前商品库存，不按时间范围回溯历史库存。

新增列表：

- 售罄 SKU：
  - `status = active` 且 `stockQty = 0`。
- 低库存 SKU：
  - `status = active` 且 `stockQty > 0` 且 `stockQty < 3`。
- 高风险 SKU：
  - 当前范围售出件数 >= 2 且当前库存 <= 2。
- 滞销 SKU：
  - 当前范围售出件数 = 0，当前库存 > 0，且 `isSellable = true`。
- 补货建议：
  - 当前范围售出件数 >= 2 且当前库存 <= 2。
  - 与高风险 SKU 可共用数据，但文案更偏行动建议。

## 6. UI/UE 设计

### 6.1 顶部时间范围控制

仪表盘顶部结构建议：

- 左侧：
  - 标题：仪表盘。
  - 当前范围说明。
- 右侧：
  - 时间范围分段按钮。
  - 自定义日期输入。
  - 刷新按钮。

在窄屏上：

- 时间范围按钮允许换行。
- 自定义日期输入上下排列。
- 刷新按钮保留在同一控制区内。

### 6.2 模块顺序

V1.4a 保持现有模块顺序：

1. 经营概览。
2. 售后概览。
3. 热销 SKU。
4. 赠品消耗。
5. 低库存 SKU。
6. 异常订单。

V1.4b 增加后建议顺序：

1. 时间范围控制。
2. 经营概览。
3. 出库与客单。
4. 售后和支付。
5. 商品表现。
6. 活动效果。
7. 库存风险。
8. 异常订单。

### 6.3 信息密度

继续保持工具型高密度界面：

- 不做大 hero。
- 不做复杂图表。
- 优先使用数字、短列表、排行和标签。
- 每个模块的空态必须是短中文。

## 7. 技术设计

### 7.1 领域层

建议重构 `src/domain/dashboard.ts`：

- 从 `day: Date` 改为 `dateRange: DashboardDateRange`。
- 新增时间范围构建函数。
- 所有统计函数统一使用 `isWithinRange(value, range)`。

建议类型：

```ts
export type DashboardRangePreset = "today" | "yesterday" | "last3days" | "last7days" | "custom";

export type DashboardDateRange = {
  preset: DashboardRangePreset;
  startAt: string;
  endAt: string;
  label: string;
};
```

### 7.2 页面层

`DashboardPage.tsx` 增加：

- `rangePreset` state。
- `customStartDate` state。
- `customEndDate` state。
- 当前范围计算。
- 自定义范围错误提示。

页面继续一次性读取当前本地数据：

- `orders`
- `orderItems`
- `products`
- `refunds`

时间范围切换只重新计算前端模型，不重复读取 Dexie。点击刷新才重新读取数据。

### 7.3 测试

- 今日范围。
- 昨天范围。
- 近 3 天范围。
- 近 7 天范围。
- 自定义范围。
- 自定义结束日期早于开始日期。
- 退款按退款时间进入范围。
- 作废按作废时间进入范围。
- 热销 SKU、赠品、异常订单按范围变化。

V1.4b 测试重点：

- 售出件数、赠品件数、总出库件数。
- 客单价。
- SPU 排名。
- 支付方式金额和订单数。
- 优惠加购件数、让利金额、参与订单数。

## 8. 不做内容

V1.4a/V1.4b 不做：

- 毛利。
- 毛利率。
- 库存成本金额。
- 库存售价金额。
- 订单明细成本快照。
- CSV 导出。
- 自定义图表。
- 拖拽仪表盘。
- 云同步。
- 多设备实时协作。

这些留到 V1.5 或以后处理。

## 9. 验收标准

### 9.1 V1.4a 验收标准

- 能切换今日、昨天、近 3 天、近 7 天和自定义日期范围。
- 跨 0 点后仍能查看昨天数据。
- 当前所有 V1.4 模块均按所选范围变化。
- 自定义日期范围错误有中文提示。
- 刷新按钮仍可用。
- 初始加载失败保护不回归。
- 全量测试和生产构建通过。

### 9.2 V1.4b 验收标准

- 能查看售出件数、赠品件数、总出库件数和客单价。
- 能查看 SPU 排名。
- 能查看支付方式订单数和金额。
- 能查看优惠加购和满赠档位统计。
- 能查看售罄、低库存、高风险、滞销和补货建议。
- 所有统计都随时间范围变化。
- 不新增不可靠毛利数据。
- 全量测试和生产构建通过。

### 9.3 V1.5 验收标准

- 新订单明细保存成本快照。
- 备份导入导出兼容成本快照。
- 毛利和毛利率按成本快照计算。
- 旧订单缺少成本快照时有明确处理策略。
- 库存价值统计口径明确。
