# ECRM V1.3c/V1.3 验收收口记录：订单售后、退款、备份与 UI 收口

## 1. 版本定位

V1.3c 是 V1.3 订单售后能力的收口版本，目标是让线下摆摊售卖中最常见的订单异常场景可以被清晰记录、复核和备份：

- 已支付订单可以作废，并记录作废原因和备注。
- 订单详情可以记录人工退款，不调用微信或支付宝真实退款接口。
- 订单记录列表可以显示作废、退款、备注等售后标识。
- 订单详情中的库存流水、售后摘要和退款记录可以被操作员快速阅读。
- 备份导入导出包含退款记录，旧备份导入后不会残留孤立退款数据。

本版本不做复合二维码，不做云端同步，不做真实支付平台退款，不做飞书式可配置复杂仪表盘。

## 2. 已完成内容

### 2.1 订单作废原因和备注

- 作废订单时需要选择作废原因。
- 支持作废备注。
- 订单详情显示作废时间、作废原因和作废备注。
- 订单记录列表显示 `已作废`、作废原因和 `有备注` 等售后标识。
- 作废后仍保留库存流水，库存回滚可追溯。

### 2.2 订单详情库存流水可读性

- 订单详情优先显示库存摘要。
- 完整库存流水保留在可展开区域。
- 空库存流水显示固定空态：`暂无库存流水。`
- 空库存流水测试备份已提供：
  - `docs/manual-test-data/ecrm-empty-inventory-order-backup.json`

### 2.3 订单售后标识

- 订单记录列表根据订单状态、作废原因、备注和退款记录生成售后标识。
- 支持显示：
  - `已作废`
  - 作废原因，例如 `客户取消`、`误操作`、`收款异常`
  - `有备注`
  - `部分退款`
  - `已退款`
- 标签 UI 已在 V1.3c-5 统一尺寸、字号、行高、圆角和颜色权重。

### 2.4 人工退款记录

- 订单详情支持 `记录退款`。
- 可录入：
  - 退款金额
  - 退款方式
  - 退款原因
  - 退款备注
- 支持对已支付订单和已作废订单记录人工退款。
- 订单详情显示累计退款和剩余可退。
- 退款金额不能超过剩余可退金额。
- 全额退款后订单记录列表显示 `已退款`。
- 部分退款后订单记录列表显示 `部分退款`。
- 退款记录不改变订单主状态，避免把人工售后和订单支付状态混淆。

### 2.5 退款失败与刷新失败处理

- 真实保存退款失败时，退款弹窗保持打开，输入内容不丢失。
- 退款保存失败只显示脱敏中文错误：`退款记录保存失败，请刷新后重试。`
- 不显示底层 raw error。
- 退款已保存但详情退款记录刷新失败时，显示：
  - `订单 订单号 已记录退款，但退款记录刷新失败，请刷新页面查看最新退款记录。`
- 退款已保存但售卖数据刷新失败时，显示：
  - `订单 订单号 已记录退款，但售卖数据刷新失败，请刷新页面查看最新订单列表。`
- 退款标识加载失败不会阻断商品、设置和订单核心数据加载。

### 2.6 备份导入导出

- 备份版本已升级到版本 3。
- 备份包含：
  - 商品
  - 图片
  - 设置
  - 订单
  - 订单明细
  - 库存流水
  - 人工退款记录
- 导入旧备份时会清空现有退款表，避免旧备份导入后残留当前环境中的孤立退款数据。
- 导出、导入备份后，订单和退款记录未发现丢失问题。

### 2.7 订单售后 UI/UE 收口

- 订单记录列表标签统一为同一套 chip 样式。
- 状态、支付方式和售后标签尺寸、字号、圆角和行高已统一。
- 订单详情售后摘要和退款记录的字号、边距、边框和信息密度已统一。
- 订单详情底部 `记录退款` 和 `作废订单` 增加独立按钮组和固定间距。
- 操作区背景改为中性信息区，仅 `作废订单` 保持危险按钮视觉。

## 5. 当前已知限制

- `cancelled` 仍同时表达取消和作废，没有新增独立 `voided` 状态。
- 人工退款只是记录，不自动调用微信或支付宝退款接口。
- 退款不会自动回滚库存；库存回滚仍由订单作废流程处理。
- 不支持订单局部商品调整、补差价、退差价或换货记录。
- 不支持云端同步，多设备同时售卖仍可能产生库存和订单冲突。
- 不支持复杂可配置仪表盘。
- 当前仪表盘只具备基础概览，需要在 V1.4 扩展为更实用的固定统计面板。
