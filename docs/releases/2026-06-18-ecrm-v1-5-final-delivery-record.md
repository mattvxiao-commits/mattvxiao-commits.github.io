# ECRM V1.5 最终交付记录

## 版本范围

V1.5 按计划完成三块能力：

- V1.5a：订单明细成本快照。
- V1.5b：仪表盘毛利统计。
- V1.5c：订单 Excel 表格导出。

本版本目标是让线下摆摊场景具备更可靠的成本、毛利、退款、库存和订单复盘能力，并在不引入云端同步的前提下，继续保持本机离线可用。

## V1.5a 成本快照

- 新订单明细会保存下单当时的单位成本快照、成本小计和明细毛利。
- 普通商品、优惠加购、赠品都写入成本快照。
- 赠品销售小计为 0，但成本仍计入，赠品会形成负毛利。
- 订单详情页可查看成本和毛利；旧订单缺少快照时显示“缺少成本快照”。
- JSON 备份升级到 version 4，并兼容旧版本备份导入。
- 旧订单不会用当前商品成本补写历史成本，避免伪造毛利。

## V1.5b 毛利仪表盘

- 仪表盘新增毛利概览：销售额（有成本快照）、成本、毛利、毛利率、赠品成本。
- 新增 SKU 毛利排行、SPU 毛利排行、低毛利 SKU。
- 毛利统计沿用时间范围切换：今日、昨天、近 3 天、近 7 天、自定义日期。
- 缺少成本快照的旧订单明细不会纳入准确毛利，并在页面提示缺失数量和关联订单。
- 人工退款仍按订单级记录，不拆分到 SKU/SPU 毛利。

## V1.5c 订单 Excel 导出

- 设置页新增“表格导出”区块，提供“导出订单 Excel”按钮。
- Excel 文件名格式为 `ecrm-orders-YYYY-MM-DD.xlsx`。
- 导出文件包含 6 张表：
  - `订单汇总`
  - `订单明细`
  - `退款记录`
  - `库存流水`
  - `商品当前数据`
  - `导出说明`
- Excel 用于统计、盘点和复盘，不能用于恢复系统数据；恢复数据仍必须使用 JSON 备份。
- 导出失败时显示统一中文错误，不暴露底层异常信息。

## 已知边界

- 历史订单缺少成本快照时，不会补算历史毛利。
- 人工退款仍是订单级记录，不拆分到 SKU/SPU 毛利。
- 当前人工退款不会自动产生退货入库或库存回补。
- 商品级退款、退货入库、换货、补差价、退差价仍属于后续版本范围。
- 无订单时 Excel 仍可导出空订单表和当前商品数据。
- 引入 `xlsx` 后生产构建出现主包体积提示，构建本身通过。离线本机使用场景当前可接受，后续可通过 `dynamic import()` 或路由级拆包优化首屏体积。
