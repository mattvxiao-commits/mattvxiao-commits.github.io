# ECRM V1.6.4 仪表盘复盘结构整理记录

## 1. 本轮目标

V1.6.4 聚焦仪表盘页面内部结构整理。目标是在不改变统计口径、不启动 V1.7 全局 UI 壳层改版的前提下，让现有仪表盘模块更适合摆摊后的快速复盘。

本轮重点解决：

- 仪表盘模块数量增加后，页面信息层级不够清晰的问题。
- 销售、活动、毛利、库存、异常信息混排导致复盘路径不明确的问题。
- 后续 V1.7 全局 UI 改版前，需要先稳定仪表盘内容结构的问题。

## 2. 已完成内容

### 2.1 新增仪表盘内部分区导航

仪表盘数据加载完成后，顶部新增“仪表盘分区”内部导航：

- 概览。
- 销售。
- 活动。
- 毛利。
- 库存。
- 异常。

该导航仅用于页面内快速跳转，不改变全局菜单，也不影响现场模式锁定逻辑。

### 2.2 新增六个复盘分区

现有仪表盘模块已按复盘用途重新归组：

- 经营概览：销售额、退款、实收、订单、售后概览、订单性质。
- 销售表现：出库与客单、支付方式、热销 SKU、热销 SPU、SPU 销售额。
- 活动成本：活动效果、非销售拆分、经营成本口径、满赠触发、非销售出库分布。
- 毛利分析：毛利概览、SKU 毛利排行、SPU 毛利排行、低毛利 SKU。
- 库存风险：赠品消耗、低库存 SKU、售罄 SKU、高风险 SKU、滞销 SKU、补货建议。
- 订单异常：异常订单。

### 2.3 保持现有统计逻辑不变

本轮只调整页面组织方式和视觉层级，不改动仪表盘领域统计模型：

- 不改变销售收入口径。
- 不改变销售成本、运营活动成本、非经营出库成本、全出库成本口径。
- 不改变日期范围筛选。
- 不改变统计口径筛选。
- 不改变订单、退款、库存流水原始数据。

### 2.4 增加页面结构测试

新增仪表盘页面测试，覆盖：

- 分区导航文案和顺序。
- 六个复盘分区标题和顺序。
- 关键模块归属到正确分区。
- 既有“经营概览”指标条与新增“经营概览”分区同名时，测试定位仍然稳定。

## 3. 设计说明

V1.6.4 采用“先整理内容结构，再做全局 UI 壳层”的节奏。

原因是当前仪表盘已经包含销售、活动、毛利、库存、异常等多类经营信息。如果直接进入 V1.7 全局 UI 改版，而仪表盘内部信息结构仍不稳定，后续会反复调整页面壳层和模块位置，开发成本和 UI 回归风险都会提高。

本轮整理后，V1.7 可以更明确地围绕以下基础继续设计：

- 全局导航如何容纳仪表盘。
- 移动端如何切换仪表盘模块。
- 仪表盘顶部筛选区和内容分区如何在小屏下保持可用。
- 现场模式下哪些复盘内容需要保护或弱化入口。

## 6. 下一步建议

建议先进行 V1.6.4 本地验收：

- 查看仪表盘分区导航是否符合复盘习惯。
- 检查各模块归组是否符合现场经营复盘逻辑。
- 确认移动端或窄屏下分区内容是否仍可阅读和操作。

验收通过后，再进入 V1.7 全局 UI 壳层改版。V1.7 建议重点处理：

- 全菜单布局和顶部区域统一。
- 移动端基础导航。
- 现场模式入口和锁定状态表达。
- 仪表盘筛选区、页面分区与小屏操作的整体统一。
