# ECRM V1.6.1 UI/UE 修复交付记录

## 版本

- 版本号：v1.6.1
- 日期：2026-06-21
- 分支：v1.6.1a-ui-ue-fixes

## 本版重点

- 压缩购物车内非销售出库快捷操作区，减少运营赠礼、人工赠送、其他出库按钮对购物车内容的挤占。
- 仪表盘筛选区改为明确分组：时间范围、刷新、自定义日期和统计口径分层展示，避免按钮散乱和刷新按钮过宽。
- 运营赠礼配置支持按 SKU 或 SPU 设置默认赠礼目标。选择 SPU 时，开单时会在该 SPU 下选择具体可赠礼 SKU。
- 保持 V1.6.0 已上线的混合订单、非销售出库、仪表盘口径、备份和导出逻辑不回退。

## 购物车体验

- 非销售出库入口统一为紧凑按钮：`+ 运营赠礼`、`+ 人工赠送`、`+ 其他出库`。
- 快捷入口保留在购物车内，方便现场开单时直接补充运营赠礼或非销售出库。
- 购物车行项目仍使用原业务标签展示，例如“运营赠礼”“人工赠送”，不把按钮符号写入订单明细。

## 仪表盘体验

- 顶部筛选区新增“仪表盘筛选”结构分组。
- 时间范围按钮和刷新按钮位于同一行，刷新按钮保持普通按钮尺寸。
- 自定义日期只在选择“自定义”后展示，避免常态占用空间。
- 统计口径单独成行，便于在正常销售、全部活动、运营赠礼、人工赠送和其他出库之间切换。

## 运营赠礼配置

- 设置页运营赠礼新增默认目标类型：SKU / SPU。
- SKU 模式延续旧逻辑，可直接配置某一个默认赠礼商品。
- SPU 模式可配置某个 SPU，开单时从该 SPU 下所有可赠礼 SKU 中选择。
- 旧备份和旧设置会自动补齐为 SKU 模式，不影响已有数据导入。

## 未纳入本版

- 暂不调整全局导航为左侧栏。
- 暂不重做整套页面布局和视觉系统。
- 暂不新增复杂活动引擎、外部平台关注验证或完整仪表盘搭建器。

## 验收建议

- 在售卖页展开购物车，检查非销售出库按钮是否明显更紧凑，购物车商品内容可见区域是否增加。
- 在设置页把运营赠礼目标类型切换为 SPU，选择一个 SPU 后，到售卖页点击运营赠礼，应弹出该 SPU 下可赠礼 SKU。
- 在仪表盘切换今日、昨天、近 3 天、近 7 天、自定义日期和统计口径，检查筛选区是否清晰、刷新按钮是否不再被拉成长条。
- 导出旧备份再导入，确认运营赠礼配置和订单数据保持兼容。
