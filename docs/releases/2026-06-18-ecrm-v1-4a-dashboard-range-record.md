# ECRM V1.4a 仪表盘时间范围切换交付记录

## 1. 版本定位

V1.4a 为仪表盘增加今日、昨天、近 3 天、近 7 天和自定义日期范围切换，解决跨 0 点后无法查看昨天或前几天摆摊数据的问题。

本版本只处理时间范围底座，不做 V1.4b 的经营统计扩展，也不做 V1.5 的成本快照、毛利、毛利率和库存价值。

## 2. 已完成内容

- 增加今日、昨天、近 3 天、近 7 天、自定义日期范围。
- 仪表盘现有经营概览、售后概览、热销 SKU、赠品消耗、异常订单均按当前范围统计。
- 低库存 SKU 继续显示当前库存，不做历史库存回溯。
- 自定义日期范围支持开始日期和结束日期。
- 自定义结束日期早于开始日期时显示 `结束日期不能早于开始日期。`。
- 自定义日期不完整或无效时不展示上一有效范围统计，避免误读。
- 每次进入自定义范围时默认重置为今天到今天。
- 时间范围切换只基于已加载数据重新计算，不重复读取本地数据库。
- 刷新按钮保留，并继续重新读取本地数据库。
- 仪表盘顶部时间范围控件完成紧凑化样式，支持小屏换行。
- 加载失败保护保持可用，错误文案不暴露底层异常。

## 3. 统计口径

- 已支付订单按 `paidAt ?? createdAt` 归属范围。
- 作废订单按 `cancelledAt` 归属范围。
- 退款金额按 `OrderRefund.createdAt` 归属范围。
- 范围内订单的部分退款、已退款状态按该订单全部累计退款判断。
- 异常订单中已作废订单按 `cancelledAt` 归属范围。
- 异常订单中退款、备注、赠品异常按已支付订单的 `paidAt ?? createdAt` 归属范围。
- 当前范围使用浏览器本地时间。
- 近 3 天和近 7 天均包含今天。
- 自定义范围从开始日期 `00:00:00.000` 到结束日期 `23:59:59.999`。

## 5. 已知限制

- 不支持用户手动设置时区，全部按当前浏览器本地时间计算。
- 自定义范围只支持日期，不支持精确到小时或分钟。
- 低库存统计仍为当前库存快照，不回溯历史库存。
- 本版本暂不新增售出件数、客单价、SPU 排行、支付方式、活动效果、库存风险扩展等 V1.4b 指标。
- 本版本暂不做毛利、毛利率、成本快照和库存价值。
