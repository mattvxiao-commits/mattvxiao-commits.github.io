# ECRM V1.4b-final 总体验收记录

## 1. 版本定位

V1.4b-final 是 V1.5 成本、毛利和订单 Excel 导出开发前的阶段性封版记录。

截至本记录，ECRM 已从轻量 MVP 扩展为可支持线下摆摊个人原创 / 同人衍生周边售卖的本地离线 CRM / POS 工具雏形。当前版本重点覆盖：

- 商品管理。
- 售卖开单。
- 购物车与收款页。
- 优惠加购与满赠。
- 订单详情、作废和人工退款记录。
- 备份导入导出。
- 固定轻量经营仪表盘。
- 时间范围切换。
- 出库、客单、SPU、支付方式、活动效果和库存风险统计。

本版本仍保持本地离线优先，不做云同步、真实支付接口、复合二维码、飞书式复杂拖拽仪表盘或自动采购任务。

## 2. 当前主线版本构成

### 2.1 V1.3c：订单售后、退款、备份与 UI 收口

已完成：

- 已支付订单作废。
- 作废原因和作废备注。
- 作废后库存回滚与库存流水保留。
- 订单详情库存摘要和完整流水展开。
- 订单记录售后标签。
- 人工退款记录。
- 部分退款和已退款标识。
- 退款保存失败与刷新失败的脱敏中文错误提示。
- 备份版本 3，包含人工退款记录。
- 旧备份导入时清理孤立退款数据。

已知边界：

- 人工退款只是记录，不调用微信或支付宝真实退款接口。
- 退款不会自动按商品明细回滚库存。
- 暂不支持商品级退款、换货、补差价或退差价。

### 2.2 V1.4：轻量实用仪表盘

已完成：

- 经营概览：销售额、退款、实收、订单数。
- 售后概览：作废订单、部分退款、已退款、作废备注。
- 热销 SKU。

- 低库存 SKU。

- 仪表盘加载失败保护。

### 2.3 V1.4a：仪表盘时间范围切换

已完成：

- 今日。
- 昨天。
- 近 3 天。
- 近 7 天。
- 自定义日期范围。
- 自定义日期错误提示。
- 时间范围切换基于已加载数据重新计算。
- 刷新按钮保留，用于重新读取本地数据库。

统计口径：

- 已支付订单按 `paidAt ?? createdAt` 归属范围。
- 作废订单按 `cancelledAt` 归属范围。
- 退款金额按 `OrderRefund.createdAt` 归属范围。
- 当前范围按浏览器本地时间计算。

### 2.4 V1.4b-1：出库与客单

已完成：

- 售出件数。
- 赠品件数。
- 总出库件数。
- 客单价。

统计口径：

- 售出件数统计 `normal` 和 `discount_addon` 明细。
- 赠品件数统计 `gift` 明细。
- 总出库件数 = 售出件数 + 赠品件数。
- 客单价 = 当前范围实收 / 当前范围已支付订单数。

### 2.5 V1.4b-2：SPU 表现

已完成：

- 热销 SPU 排行。
- SPU 销售额排行。

统计口径：

- 只统计当前范围已支付订单。
- 只统计 `normal` 和 `discount_addon` 明细。
- 不统计 `gift` 赠品行。
- 按订单明细 `spuSnapshot` 聚合。
- 每个排行最多展示 5 条。

### 2.6 V1.4b-3：支付方式与活动效果

已完成：

- 微信、支付宝、现金、其他、未记录支付方式统计。
- 支付方式订单数和金额。
- 优惠加购件数。
- 优惠让利金额。
- 参与优惠订单数。
- 满赠触发订单数。
- 满赠档位触发统计。

统计口径：

- 支付方式金额使用订单 `payableAmount`，不扣退款。
- 退款拆分回支付方式留给后续版本。
- 优惠加购只统计 `discount_addon` 明细。
- 满赠触发按订单 `triggeredGiftTier` 聚合。

### 2.7 V1.4b-4：库存风险

已完成：

- 售罄 SKU。
- 低库存 SKU。
- 高风险 SKU。
- 滞销 SKU。
- 补货建议。

统计口径：

- 库存风险使用当前商品库存，不按日期回溯历史库存。
- 售罄 SKU 为启用且库存为 0 的商品。
- 低库存 SKU 为启用、库存大于 0，且满足当前库存少于 3 或估算剩余比例不高于 20% 的商品。
- 高风险 SKU 为当前范围售出件数大于等于 2，且库存小于等于 2 或估算剩余比例不高于 20% 的启用可售商品。
- 滞销 SKU 为当前范围无销量、当前库存大于 0、启用且可售的商品。
- 补货建议优先展示可售售罄 SKU，其次展示高风险 SKU。
- 估算剩余比例 = 当前库存 /（当前库存 + 当前范围售出件数）。
- 估算剩余比例不等于真实历史期初库存比例。

## 3. 当前用户验收结论

用户已在本地界面持续验收并确认：

- 商品新增、编辑、复制创建相似 SKU 功能符合预期。
- 商品图片可正常显示。
- 售卖页紧凑列表、图片网格切换和购物车收缩体验符合当前阶段预期。
- 商品表单必填项提示、购物车持久化、备份导入覆盖风险提示、图片恢复等问题已处理。
- 商品管理大弹窗和复制功能已验收通过。
- 收款页二维码区域和多余文案已优化。
- 订单详情信息完整，订单作废和多原因选择生效。
- 空库存流水、备份导入导出、退款功能、售后 UI/UE 修复均已验收通过。
- 仪表盘时间范围切换、经营统计、SPU 统计、支付方式和活动效果统计已陆续验收。
- 库存风险统计已根据反馈完成口径调整，20% 估算剩余比例阈值已落地。

当前结论：

V1.4b-final 可以作为 V1.5 成本、毛利和订单 Excel 导出开发前的稳定功能基线。

## 4. 当前已知限制

### 4.1 设备与部署

- 当前尚未完成 Windows、Mac 和 iPad 的跨设备完整验收。
- 当前尚未完成 GitHub 仓库创建、远程推送、版本 tag 推送和跨设备部署说明。
- 当前没有云同步，多设备同时售卖仍可能产生库存和订单冲突。
- iPad 作为 PWA 使用仍需要 HTTPS 访问路径或其他部署方案。

### 4.2 支付与退款

- 不接入真实微信或支付宝支付 API。
- 不做支付平台到账核验。
- 人工退款不调用真实支付平台退款接口。
- 退款金额暂不拆分回具体支付方式。
- 退款暂不按商品级明细拆分。
- 退款不自动退回库存，库存回滚仍由订单作废流程处理。

### 4.3 订单调整

- 暂不支持商品级换货。
- 暂不支持补差价、退差价。
- 暂不支持订单局部商品调整。
- 暂不支持退款退货入库。

### 4.4 仪表盘与经营分析

- 当前仪表盘是固定轻量统计，不支持拖拽式图表配置。
- 当前不支持飞书式复杂多维仪表盘。
- 当前不支持 CSV 或 Excel 表格导出。
- 当前不包含成本快照、毛利、毛利率和库存价值统计。
- 当前库存风险使用当前库存快照，不回溯历史库存。

### 4.5 数据与备份

- JSON 备份用于恢复系统数据，尚未提供面向 Excel 的表格化订单导出。
- 备份导入会覆盖当前本地数据，虽然已有风险确认，但仍需要操作员保持导入前导出备份习惯。
- 浏览器本地 IndexedDB 仍是当前主要数据存储，清浏览器数据可能导致业务数据丢失。

3. V1.5b 毛利、毛利率和成本相关仪表盘统计。

5. V1.5-final 总体验证、分支合并准备和正式版本 tag。

### 5.1 V1.5a 订单成本快照

目标：

- 新订单下单时，在订单明细中保存商品成本快照。
- 避免未来修改商品成本价后影响历史订单毛利。
- 为 V1.5b 毛利统计和 V1.5c Excel 导出提供可靠数据基础。

建议字段：

- 单位成本快照。
- 成本小计。
- 明细毛利。

建议口径：

- 普通商品、优惠加购商品和赠品都保存成本快照。
- 赠品销售小计为 0，但成本小计应计入赠品成本。
- V1.5a 前的历史订单不伪造真实成本快照，后续页面和导出中标记为缺少成本快照。

### 5.2 V1.5b 毛利统计

目标：

- 在仪表盘增加固定毛利统计模块。
- 让用户能查看当前范围内真实可追溯的成本、毛利和毛利率。

建议统计：

- 当前范围销售额。
- 当前范围成本。
- 当前范围毛利。
- 当前范围毛利率。
- SKU 毛利排行。
- SPU 毛利排行。
- 低毛利 SKU。
- 赠品成本。
- 优惠让利与赠品成本合计。

建议边界：

- 默认只把有成本快照的订单明细计入准确毛利。
- V1.5a 前缺少成本快照的历史订单不纳入准确毛利统计。
- 人工退款暂不按商品级成本拆分，商品级退款和退货入库留给后续订单调整版本。

### 5.3 V1.5c 订单记录 Excel 导出

目标：

- 新增订单记录单独导出功能。
- 导出可由 Excel 打开的 `.xlsx` 表格文件。
- 用于整体表格化统计、盘点和复盘。

建议导出内容：

- 订单汇总。
- 订单明细。
- 退款记录。
- 库存流水。
- 商品当前数据。
- 导出说明。

建议页面位置：

- 先放在设置页，靠近备份与恢复区域。
- 文案明确区分：
  - JSON 备份用于恢复系统数据。
  - Excel 导出用于统计、盘点和复盘，不能用于恢复系统。
