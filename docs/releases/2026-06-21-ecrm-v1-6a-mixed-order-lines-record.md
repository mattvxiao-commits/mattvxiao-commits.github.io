# ECRM V1.6a 混合订单与赠送明细交付记录

## 版本

- 版本号：v1.6.0
- 日期：2026-06-21
- 分支：v1.6a-mixed-order-lines

## 本版重点

- 支持同一订单内同时记录正常销售商品、运营赠礼、人工赠送和其他非销售出库商品。
- 非销售明细自动按 0 元处理，不计入应收金额，但会扣减库存并保留成本快照。
- 新增运营赠礼配置，适合记录关注社媒、加入社群、现场互动等活动产生的赠品出库。
- 支持纯人工赠送、其他非销售出库，不再需要用“其他收款 + 退款”的方式绕路记录。
- 订单详情支持整单和明细级统计口径修正，历史支付、退款和库存流水保持原始事实。
- 仪表盘新增统计口径切换，默认展示正常销售数据，避免赠礼和非销售出库污染销售排行、客单价和毛利。
- 订单 Excel 导出增加 V1.6a 明细口径、成本归属、经营归属和统计修正字段。
- JSON 备份升级到 version 5，兼容 V1.6a 明细字段和非销售出库库存流水。

## 业务口径

- 正常销售：计入销售额、客单价、销售排行、销售成本和基础销售毛利。
- 加购优惠销售：仍属于销售，商品成本归销售成本，原价与成交价差额单独记为优惠让利。
- 满赠赠品：计入满赠成本，归属运营活动成本。
- 运营赠礼：计入运营赠礼成本，归属运营活动成本。
- 人工赠送：计入人工赠送成本，归属非经营赠送。
- 其他非销售出库：计入其他非销售出库成本，单独用于库存和异常出库复盘。

## 仪表盘变化

- 默认口径为“正常销售”，只统计销售明细。
- “全部活动”口径展示销售、赠礼和非销售出库的完整成本消耗。
- 支持单独查看运营赠礼、人工赠送、其他出库口径。
- 新增优惠让利、销售成本、运营活动成本、活动后毛利、非经营出库、全出库成本指标。
- 非销售出库分布用于查看赠礼和非销售出库 SKU 消耗。

## 导出和备份

- 订单明细导出新增：
  - 明细收入类型。
  - 非销售原因。
  - 非销售备注。
  - 运营活动名称快照。
  - 是否统计修正。
  - 原始销售小计。
  - 统计销售小计。
  - 优惠让利金额。
  - 成本归属。
  - 经营归属。
  - 统计修正时间和备注。
- 订单汇总导出新增：
  - 订单整体性质。
  - 销售金额。
  - 优惠让利。
  - 销售成本。
  - 基础销售毛利。
  - 运营活动成本。
  - 活动后毛利。
  - 非经营出库成本。
  - 非销售出库件数。
  - 非销售出库成本。
  - 是否含统计修正。
- JSON 备份当前版本为 version 5。
- 旧版本备份仍可导入，旧订单明细由系统读取时按兼容口径归一化，不强行改写历史原始数据。

## 兼容说明

- 旧订单默认按正常销售读取。
- 旧满赠赠品可通过 `lineType = gift` 归一化为满赠赠品。
- 历史统计修正只影响统计口径，不删除或重写原始支付、退款和库存流水。
- Excel 文件用于复盘和盘点，不用于恢复系统数据；恢复数据仍使用 JSON 备份。

## 已知边界

- 不自动验证小红书、微博、社群等外部平台的关注或互动状态。
- 不接入微信、支付宝支付 API，也不自动查账。
- 不实现商品级退款、退货入库、换货和补差价。
- 不实现云同步或多设备自动合并。
- 不实现完整活动引擎或拖拽式仪表盘。

## 建议验收路径

- 消费商品 + 运营赠礼同单下单，应收只包含销售商品，库存同时扣减销售商品和赠礼。
- 纯人工赠送下单，不要求选择支付方式，保存后扣库存。
- 其他非销售出库下单，必须填写原因，保存后进入订单记录。
- 历史订单整单修正为人工赠送，仪表盘正常销售口径应排除该订单。
- 历史订单单行修正为运营赠礼，原始支付和库存流水不应被删除。
- 仪表盘切换正常销售、全部活动、运营赠礼、人工赠送、其他出库口径，指标应随口径变化。
- 导出订单 Excel，检查“订单明细”和“订单汇总”是否包含 V1.6a 字段。
- 导出 JSON 备份再导入，订单、非销售明细、库存流水和图片应保留。
