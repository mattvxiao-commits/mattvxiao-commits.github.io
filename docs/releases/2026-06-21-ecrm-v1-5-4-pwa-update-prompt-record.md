# ECRM V1.5.4 PWA 更新提示交付记录

## 版本目标

`v1.5.4` 用于补齐线上部署后的 PWA 更新提示能力，减少“GitHub Pages 已部署成功，但当前设备仍看到旧版本”的不确定体验。

本版本不改变商品、订单、库存、备份、仪表盘、现场模式和数据库结构。

## 已更新

- PWA 注册方式从后台自动更新调整为显式提示更新。
- 新增“发现新版本”顶部提示条。
- 用户可点击“刷新更新”应用新版本资源。
- 用户可点击“稍后”关闭提示，避免现场操作时被遮挡。
- README 增加 PWA 更新和旧缓存处理说明。
- GitHub 跨设备说明文档增加版本更新与缓存处理章节。
- 项目版本号更新为 `1.5.4`。

## 数据安全说明

- 更新提示只处理应用静态资源。
- IndexedDB 中的商品、订单、库存、收款码、设置和备份数据不会因点击“刷新更新”被清空。
- 仍建议摆摊前、摆摊中和收摊后导出 JSON 备份。

## 验收重点

- 未检测到新版本时不显示提示条。
- 检测到新版本时显示“发现新版本”。
- 点击“稍后”可关闭提示条。
- 点击“刷新更新”会调用 PWA 更新流程。
- 页面布局不遮挡顶部导航、商品列表、购物车或收款码。

## 已知边界

- 不提供版本历史列表。
- 不强制用户立即刷新。
- 不自动清除浏览器缓存或站点数据。
- iPad 主屏幕 Web App 的更新仍受 Safari / iPadOS 的 PWA 缓存机制影响，必要时需要联网打开 Safari 中的同一地址辅助触发更新。
