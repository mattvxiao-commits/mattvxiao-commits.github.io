# ECRM V1.5.2 现场售卖隐私锁交付记录

## 版本目标

解决现场售卖时设备转向顾客扫码、误触顶部导航后可能暴露商品成本、库存、经营统计、设置和备份入口的问题。

## 已实现

- 新增现场模式 4 位数字 PIN。
- 商品、设置、仪表盘作为管理页面统一受锁保护。
- 售卖页保持可用，不影响开单、加购、购物车和收款流程。
- 输入正确 PIN 后临时解锁 5 分钟。
- 售卖页可在临时解锁期间立即重新锁定。
- 连续 5 次错误 PIN 后延迟 30 秒再允许继续尝试。
- PIN 使用盐和哈希保存，不保存明文。
- JSON 备份导出不包含 PIN 哈希、盐、临时解锁状态或错误次数。
- JSON 备份导入后现场模式默认关闭，目标设备需要重新设置 PIN。
- README 已同步当前退款、图片备份、Excel 导出、仪表盘和现场模式状态。

## 未实现

- 不做多用户账号或角色权限。
- 不做云端认证或远程找回 PIN。
- 不做数据库全量加密。
- 不做顾客全屏扫码展示模式。
- 不承诺抵御具备浏览器开发者工具能力的技术攻击者。

## 手动验收建议

- 打开设置页，输入相同的 4 位 PIN，开启现场模式并保存。
- 回到售卖页，确认售卖商品列表和购物车仍可正常使用。
- 点击顶部“商品”，确认先出现“管理页面已锁定”弹窗。
- 输入错误 PIN，确认不能进入商品页。
- 输入正确 PIN，确认可以进入商品页。
- 回到售卖页，点击“重新锁定”，再次点击商品，确认需要重新输入 PIN。
- 点击顶部“设置”和“仪表盘”，确认同样需要 PIN。
- 导出 JSON 备份，确认备份文件中不包含 `pinHash`、`pinSalt`。
- 导入备份到新设备或清空数据后导入，确认现场模式默认关闭。
