# ECRM V1.5c 订单 Excel 导出交付记录

## 交付范围

- 设置页新增“表格导出”区块，提供“导出订单 Excel”按钮。
- Excel 导出与 JSON 备份/恢复分离，页面明确提示：Excel 用于统计、盘点和复盘，不能用于恢复系统数据；恢复数据请使用 JSON 备份。
- 导出文件名格式为 `ecrm-orders-YYYY-MM-DD.xlsx`，日期来自本次导出的 `exportedAt`。
- Excel 文件包含 6 张表：
  - `订单汇总`
  - `订单明细`
  - `退款记录`
  - `库存流水`
  - `商品当前数据`
  - `导出说明`
- 订单 Excel 导出成功后显示“订单 Excel 已导出。”。
- 订单 Excel 导出失败时显示“订单 Excel 导出失败，请稍后重试。”，不向界面暴露底层错误信息。

## 导出口径

- `订单汇总` 导出订单编号、订单状态、支付方式、创建/支付/作废时间、原价合计、优惠金额、应收金额、退款金额、实收估算、满赠档位和作废信息。
- `订单明细` 导出商品快照、SPU、商品编码、数量、明细类型、原单价、成交单价、销售小计、成本快照、成本小计、毛利、毛利率和是否缺少成本快照。
- `退款记录` 导出订单级人工退款记录。V1.5c 不把人工退款拆分到 SKU/SPU 毛利。
- `库存流水` 导出订单扣减、赠品扣减、作废回滚、人工调整等库存变动。
- `商品当前数据` 导出当前商品库数据，用于盘点时对照当前库存、成本价、售价、可售状态和赠品状态。
- `导出说明` 写入导出时间、系统版本、文件用途、恢复说明、成本快照说明、旧订单说明、人工退款说明和退款退货入库说明。

## 成本与历史数据说明

- 成本字段来自订单明细在下单时保存的成本快照。
- 旧订单缺少成本快照时，Excel 成本和毛利字段留空，并标记“是否缺少成本快照”。
- 旧订单不会使用当前商品成本补写历史成本，避免伪造历史毛利。
- 赠品明细销售小计为 0，但成本仍按成本快照导出，因此赠品成本会降低毛利。

## 退款与库存边界

- 当前人工退款是订单级记录，Excel 会导出退款金额、退款方式、退款原因和备注。
- 当前人工退款不会自动产生退货入库或库存回补。
- 如需商品级退款、退货入库、换货、补差价或退差价，仍属于后续版本范围。

## 已知边界与后续优化

- 当前导出按订单并发读取订单明细和库存流水，对本机 MVP 和摊位订单量可接受；后续订单量较大时可增加批量读取仓储 API 或限制并发。
- 无订单时仍会导出包含商品当前数据和说明表的 Excel 文件；后续可增加“暂无订单，已导出空模板/当前商品数据”的专门提示。
- 正式合并前已将 Excel 写出依赖从 `xlsx` 替换为 `write-excel-file`，规避 `xlsx` 生产依赖审计风险。
- Excel 写出依赖仍会增加前端包体积；后续可通过 `dynamic import()` 或路由级拆包把 Excel 依赖从首屏主包中拆出。
