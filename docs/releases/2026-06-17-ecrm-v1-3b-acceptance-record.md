# ECRM V1.3b 验收记录：订单作废与库存回滚

## 1. 版本定位

V1.3b 是订单管理能力的稳定小版本，目标是让现场售卖中已经误确认、重复保存、客户临时取消或需要撤销的已支付订单，可以被整单作废，并且真实回滚本订单扣减的库存。

本版本不做真实支付平台退款，不做部分退款，不做订单调整表，不升级 Dexie schema，不升级备份版本。

## 2. 已完成内容

### 2.1 订单详情作废入口

- 已支付订单详情显示 `作废订单` 操作。
- 点击作废前展示二次确认弹窗。
- 已取消订单不再显示作废入口。
- 待支付订单不显示作废入口。
- 作废操作不包含退款入口，避免现场操作员误解为自动退款。

### 2.2 订单作废事务

- 仅允许 `paid` 订单作废。
- 作废后订单状态更新为 `cancelled`。
- 写入 `cancelledAt`。
- 不能重复作废同一订单。
- 不能作废不存在、非已支付、缺少可回滚库存流水、库存流水异常或商品缺失的订单。
- 作废逻辑在 repository 层事务中完成，避免订单状态、库存、库存流水出现部分写入。

### 2.3 库存回滚

- 作废时按原始库存扣减流水回滚，而不是重新计算购物车或读取当前促销规则。
- 普通商品扣减流水 `order_paid` 会生成对应的回滚流水。
- 赠品扣减流水 `gift_order_paid` 会生成对应的回滚流水。
- 新增库存流水原因 `order_cancelled_rollback`，中文显示为 `作废回滚`。
- 回滚库存以当前库存为基准增加，能正确处理订单之后又发生其他售卖的情况。

### 2.4 订单详情库存流水可读性

- 库存流水不再优先显示难以识别的内部 `productId`。
- 现在根据订单商品快照显示：
  - 商品名称
  - 商品编码
  - SPU
  - 库存原因
  - 扣减/增加数量
  - 库存变化
- 作废后在库存流水区域展示摘要：`已作废：N 个商品库存已回滚。`
- 真实库存流水仍完整保留，不在数据层合并，保证后续对账和排查可追溯。

### 2.5 售卖页作废后刷新

- 作废成功后订单详情更新为 `已取消`。
- 作废成功后刷新订单明细、库存流水、订单列表和商品库存。
- 作废事务成功但详情刷新失败时，不再误报 `订单作废失败`。
- 此时显示：`订单 订单号 已作废，但详情刷新失败，请刷新页面查看最新库存流水。`

### 2.6 备份兼容

- 备份导入校验允许 `order_cancelled_rollback`。
- 作废后的数据可以继续导出和导入。
- 本版本不升级备份版本号。

### 2.7 去收款页 UI 修复

- 修复 checkout 模式下顶部 `刷新` 按钮被拉伸成超长按钮的问题。
- checkout 模式下刷新按钮保持右对齐、紧凑宽度。

## 5. 已知限制

- `cancelled` 同时表达“已取消/已作废”，没有单独新增 `voided` 状态。
- 不记录作废原因。
- 不记录人工退款。
- 不支持部分退款。
- 不支持订单调整。
- 不支持自动调用微信或支付宝退款。
- 仪表盘尚未按作废/退款/调整口径重构统计。

这些限制会进入 V1.3c/V1.3d 继续处理。
