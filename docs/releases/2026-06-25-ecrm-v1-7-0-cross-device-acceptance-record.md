## 5. 已记录但不阻塞的问题

跨设备验收后仍有一些 UI/UE 规范问题，但不影响 V1.7.0 发布：

- 订单详情页内部分文字规格仍需统一。
- “修正统计口径”等按钮尺寸和文案需要统一。
- 不同页面的刷新按钮样式、尺寸和位置需要统一。
- 部分页面模块间距、字体大小、按钮高度需要按统一规格复核。

这些问题不在 V1.7.0 发布后立即处理。后续建议先进入 V1.8 运营活动 / 促销规则模块化，待业务结构稳定后，再做全局 UI/UE 规格统一审查。

## 6. 后续计划

V1.7.0 发布闭环完成后，推荐继续按以下顺序推进：

1. V1.8 运营活动 / 促销规则模块化。
2. 订单、购物车、数据页、导出和备份对活动模块的联动适配。
3. 全局 UI/UE 规格统一审查。
4. 移动端专项适配。

V1.8 阶段应优先解决活动配置从“写死规则”变成“模板 + 规则引擎”的问题。旧订单兼容、订单筛选、订单详情、统计口径修正、仪表盘筛选、Excel 导出和 JSON 备份，都需要一起纳入方案设计。
