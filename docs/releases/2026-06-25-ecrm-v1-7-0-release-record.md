# ECRM V1.7.0 发布准备与本地验收记录

日期：2026-06-25  
分支：`v1.7-hybrid-full-prototype`  
目标版本：`v1.7.0`

## 1. 本轮目标

V1.7.0 是一次结构性版本更新，重点是把 ECRM 从顶部导航的早期界面，整理为更适合现场使用的 App Shell 结构。桌面端使用左侧窄导航，窄屏和移动端提供基础底部导航。订单记录从售卖页拆出为独立页面，商品、售卖、订单、数据、设置的入口更清楚。

本轮不把移动端专项体验视为完成，也不处理完整的运营活动规则模块化。移动端专项和活动规则模块会在后续版本继续推进。

## 2. 已纳入 V1.7.0 的主要内容

### 2.1 App Shell 与导航结构

- 桌面端改为左侧窄导航。
- 窄屏和移动端增加基础底部导航。
- 现场模式入口整合到导航区域，只作为状态入口，不新增独立页面。
- 现场模式支持三种状态：未锁定、临时解锁、现场模式。

### 2.2 商品与售卖

- 商品新增 / 编辑弹窗保留系列、SPU 编码、SKU 编码和完整商品编码。
- 新增“系列（筛选）”字段，用于多 IP、多系列商品的现场筛选。
- 售卖页支持系列筛选与 SPU 筛选联动。
- 修复商品完整编码重复时错误提示被弹窗遮挡的问题，现在错误会显示在商品弹窗内部。

### 2.3 订单页

- 订单记录拆分为独立订单页。
- 桌面端订单列表使用双列排布。
- 订单按时间从新到旧排列，最新订单位于第一行左侧。
- 支付方式以标签形式展示，不同支付方式使用不同颜色。
- 订单性质标签会跟随整单或单商品统计口径修正后的结果更新。
- 修复订单详情和统计修正弹窗在窄屏下底部按钮不可达的问题。

### 2.4 数据页

- 数据页沿用 V1.6.4 的复盘分区结构。
- 继续保留经营概览、销售表现、活动成本、毛利分析、库存风险和订单异常。
- 窄桌面宽度下，时间范围筛选和统计口径筛选会提前换行，避免挤压在标题右侧。

### 2.5 设置页与现场模式

- 现场模式支持锁定范围多选。
- 锁定范围包括商品页、订单详情、数据页和设置页。
- 设置页现场模式说明补充“临时解锁”含义和重新锁定路径。
- 修复导入备份或父级设置刷新后，锁定范围勾选状态不同步的问题。

### 2.6 窄屏基础可用性

- 售卖页购物车、收款面板和浮动购物车按钮避让底部导航。
- 订单详情弹窗在移动壳层下保留可滚动空间。
- 商品、数据、设置页在 900px 以下提前切换顶部区域排布。
- 极窄屏商品卡片操作按钮改为单列，避免触控目标过挤。

## 3. 本地验收结论

本地验收未发现阻断性问题。当前可进入 V1.7.0 发布准备流程。

已确认通过的重点路径：

- 商品新增、编辑、复制、系列字段和商品编码结构。
- 售卖页系列筛选和 SPU 筛选。
- 购物车、收款页、运营赠礼、人工赠送、其他出库。
- 独立订单页、订单详情、作废、退款、统计口径修正。
- 数据页筛选和复盘分区。
- 设置页现场模式、锁定范围、备份导入导出、订单 Excel 导出。

## 4. 已记录但不阻塞发布的问题

以下问题属于 UI/UE 规范、文字规格和组件一致性问题，不作为 V1.7.0 发布前阻断项：

- 订单详情页内部分文字规格、按钮尺寸和按钮文案仍需统一。
- “修正统计口径”等操作按钮需要在后续全局 UI/UE 审查中统一尺寸和视觉层级。
- 不同页面的刷新按钮样式、尺寸和位置需要统一。
- 部分页面模块间距、字体大小、按钮高度仍需要按统一规格复核。

这些问题建议归入 V1.8 活动模块之后的“全局 UI/UE 规格统一审查”阶段。原因是 V1.8 会继续改变购物车、设置页、订单详情和数据页结构，过早精修会增加返工。

## 5. 发布前验证要求

发布前必须完成：

```text
npm test -- --run
npm run build
git diff --check
```

建议发布后补充线上验收：

- GitHub Pages 页面显示版本 `v1.7.0`。
- 设置页系统信息显示版本 `v1.7.0`。
- iPad 或其他设备重新打开后能看到新版导航结构。
- 旧数据仍保存在当前设备浏览器 IndexedDB 中。
- JSON 备份可在其他设备导入，商品图、订单、库存和设置可恢复。

## 6. 后续建议

V1.7.0 发布后，建议按以下顺序继续：

1. 线上部署和跨设备安装更新测试。
2. V1.8 运营活动 / 促销规则模块化。
3. 购物车、订单、数据页、导出和备份对活动模块的联动适配。
4. 全局 UI/UE 规格统一审查。
5. 移动端专项适配。
