# ECRM V1.5.3 UI/UE 紧凑化与页头统一优化方案

## 背景

V1.5.2 已完成现场模式、订单调整、仪表盘增强、导入导出与 GitHub Pages 跨设备使用能力。当前功能可用性已经满足现场售卖基础需求，但多个核心页面仍存在开发期遗留的 UI/UE 密度和规格不统一问题：

- 商品、售卖、仪表盘、设置页的页头高度、按钮尺寸、说明文字位置不统一。
- 部分信息区块内部对齐不稳定，例如商品统计条数字明显偏上。
- 售卖页 SPU 筛选和视图按钮抢占同一横向空间，SPU 较多时筛选不便。
- 设置页现场模式卡片高度浪费明显，状态提示占用空间过大。
- 顶部品牌副标题仍是固定文案，不能反映当前店铺名称。

本轮作为 V1.5.3 的 UI/UE 统一优化，不新增复杂业务能力，目标是提高现场操作效率、首屏信息密度和视觉一致性。

## 目标

1. 建立统一的工具型页面页头规格。
2. 收敛按钮、筛选 chip、统计条、设置卡片的尺寸和间距。
3. 优化商品、售卖、仪表盘、设置四个页面中截图反馈的高优先级体验问题。
4. 让顶部品牌副标题在保存店铺名称后同步显示当前店铺名。
5. 保持现有功能和数据逻辑不变，避免引入大范围重构。

## 非目标

- 不重做整体主题色、品牌视觉或图标系统。
- 不引入新的 UI 组件库。
- 不重构购物车、订单、促销、仪表盘数据算法。
- 不增加新的业务页面。
- 不处理 PWA 更新提示，本项另列后续任务。

## 公共 UI 规格

### 页面页头

商品、售卖、仪表盘、设置页统一采用：

- 左侧：eyebrow、H1、说明文字。
- 右侧：当前页面主操作或工具组。
- 桌面端左右排列，窄屏端上下堆叠。
- H1 统一为紧凑标题规格，避免不同页面标题视觉重量差异过大。
- 说明文字保持一行到两行，字号较小，不抢主操作视觉。

### 控件尺寸

- 小型工具按钮：用于 SPU 筛选、周期切换、刷新、视图工具，目标高度约 32-34px。
- 标准按钮：用于保存、开启、导出等常规动作，目标高度约 38-40px。
- 强操作按钮：用于收款确认、购物车主操作等现场关键动作，保持更醒目。

### 卡片和区块

- 卡片圆角继续保持 8px。
- 设置页卡片 padding 收敛，减少大块空白。
- 状态信息优先使用 chip 或 inline status，避免单独占用大高度。

## 页面调整

### 商品管理

问题：

- 商品统计条内部数字和说明文字偏上，未与标题形成稳定的视觉中心。
- 顶部工具栏在宽屏下可进一步压缩，提升首屏商品可见数量。

调整：

- 商品标题、统计条、排序、新增按钮统一为紧凑工具栏布局。
- 统计条内部改为垂直居中，数字和标签使用稳定行高。
- 商品列表区域上方间距适度减少。

验收：

- 统计条数字不再贴上边。
- 商品页首屏列表可见数量不低于当前版本。
- 排序和新增按钮仍清晰可点击。

### 售卖管理

问题：

- SPU 数量较多时，横向筛选空间不足。
- 图片网格切换入口在当前现场售卖阶段不是高频能力，占用顶部空间。

调整：

- SPU 筛选独占一行横向空间。
- 顶部工具区不再把 SPU 筛选、视图切换、刷新硬挤在同一行。
- 默认隐藏图片网格切换入口，保留内部代码能力，不作为主界面入口展示。
- 刷新按钮保留在右侧或工具区，尺寸与小型工具按钮一致。

验收：

- SPU 多时仍可横向滚动筛选，显示区域明显增加。
- 主界面不再展示图片网格切换按钮。
- 紧凑列表仍为默认售卖模式。

### 仪表盘

问题：

- 周期按钮和刷新按钮尺寸、文字比例需要与其他页面工具按钮统一。

调整：

- 今日、昨天、近 3 天、近 7 天、自定义、刷新按钮统一为小型工具按钮规格。
- 自定义日期输入区展开时不挤乱标题区。

验收：

- 周期切换按钮视觉比例更紧凑。
- 自定义日期范围仍可正常选择和刷新。

### 设置

问题：

- 保存设置按钮与左侧标题区视觉不匹配。
- 现场模式卡片高度浪费明显。
- 现场模式状态提示占用单独行，信息密度低。
- 卡片右侧存在明显空白。

调整：

- 设置页页头对齐公共页头规格，让保存设置按钮与标题组垂直协调。
- 现场模式状态改为标题行右侧 chip：
  - 未开启
  - 已开启
  - 已开启 · 临时解锁
- 现场模式说明文字保持一行。
- PIN 输入和操作按钮压缩到更紧凑的工作区。
- 未开启时只展示开启现场模式按钮。
- 已开启时展示更新 PIN、立即重新锁定、关闭现场模式按钮。

验收：

- 现场模式卡片高度明显低于当前版本。
- 状态表达清晰且不单独占用大块高度。
- 开启、更新、重锁、关闭功能保持即时保存和原有锁定逻辑。

## 顶部品牌副标题同步店铺名称

当前：

- 顶部左侧固定显示 `ECRM` 和 `Booth POS 摊位工具`。

调整：

- 主标题 `ECRM` 保持不变。
- 副标题优先显示设置中的 `shopName`。
- `shopName` 为空或未加载时显示默认 `Booth POS 摊位工具`。
- 保存设置、导入备份、现场模式保存后，顶部副标题随全局 settings 更新同步刷新。

验收：

- 设置页修改店铺名称并保存后，顶部副标题立即显示新店铺名。
- 刷新页面后仍显示已保存店铺名。
- 未设置店铺名时仍显示默认副标题。

## 测试计划

自动化测试：

- App 测试覆盖顶部副标题从 settings 读取并在 settings update 事件后刷新。
- SettingsPage 测试保持现场模式即时保存、基础信息保存、导入备份等既有测试通过。
- SalesPage 测试调整为不再查找图片网格切换入口，并确认紧凑列表仍默认显示。
- DashboardPage 测试保持周期切换和数据统计可用。

构建验证：

- `npm test`
- `npm run build`
- `git diff --check`

人工验收：

- 桌面宽屏。
- 桌面非全屏 60-80% 宽度。
- iPad 或窄屏模拟。
- 重点查看商品页统计条、售卖页 SPU 筛选、仪表盘周期按钮、设置页现场模式卡片。

## 风险与控制

- 风险：CSS 公共样式影响多个页面。
  - 控制：每次只改相关选择器，避免重写全局 reset。
- 风险：隐藏图片网格入口影响已有测试。
  - 控制：更新测试预期，保留内部视图代码，后续可再开放入口。
- 风险：顶部副标题读取 settings 增加 App 初始加载状态。
  - 控制：未加载或为空时显示默认文案。
- 风险：设置页现场模式卡片压缩后按钮拥挤。
  - 控制：桌面横向布局，窄屏自动换行。

## 建议实施顺序

1. 顶部品牌副标题同步店铺名称。
2. 售卖页隐藏图片网格入口并调整筛选工具区。
3. 商品页统计条和页头对齐。
4. 仪表盘周期按钮统一。
5. 设置页现场模式卡片紧凑化。
6. 全量测试、构建、截图验收、发布。
