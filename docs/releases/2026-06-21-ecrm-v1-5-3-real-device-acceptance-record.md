## 重点验收范围

### 1. 线上访问与 PWA

- GitHub Pages 提供 HTTPS 访问。
- iPad 不需要 Apple 开发者账号、签名、TestFlight 或 App Store。
- 当前方案基于 Safari 打开 HTTPS 页面并添加到主屏幕。
- 离线可用前提是目标设备已在线打开过应用，且浏览器未清除该站点数据。

### 2. 跨设备数据迁移

- 当前没有云同步。
- 跨设备迁移依赖 JSON 全量备份。
- 备份包含商品、设置、促销配置、订单、订单明细、库存流水、退款记录、商品图和收款码图片。
- 订单 Excel 导出仅用于盘点和复盘，不能用于恢复系统数据。
- 现场模式 PIN 不随备份导出，目标设备导入后需重新设置 PIN。

### 3. 现场售卖链路

- 商品录入、复制、编辑和停用可用。
- 售卖页紧凑列表、SPU 筛选、购物车、暂存和刷新恢复可用。
- 收款页可展示微信和支付宝收款码。
- 手动确认收款后生成订单并扣减库存。
- 作废订单和人工退款记录可用，订单详情保留相关备注与状态。

### 4. 经营复盘

- 仪表盘支持今日、昨天、近 3 天、近 7 天和自定义日期范围。
- 仪表盘展示销售额、订单数、客单价、成本、毛利、毛利率、SKU/SPU 排名、支付方式、活动效果和库存风险。
- 缺少成本快照的历史订单不会被当前商品成本回填，页面会提示缺失情况。
- 订单 Excel 导出用于表格化统计、盘点和复盘。

### 5. 现场模式

- 商品、设置、仪表盘需要 PIN 解锁后进入。
- 售卖页保持可用，不影响现场开单。
- 售卖页可立即重新锁定。
- 订单记录总览可展开查看。
- 订单详情、库存流水和退款等敏感内容需要 PIN 解锁后查看。
- 解锁后会临时放行 5 分钟。

## 当前已知边界

- 不支持云同步，多设备不会自动合并订单、库存和退款记录。
- 不支持复合支付二维码。
- 不接入微信或支付宝支付 API，不自动查账。
- 不支持飞书式拖拽自定义仪表盘。
- 不支持小票打印、扫码枪等外设。
- 不支持多用户账号、角色权限或云端认证。
- 当前退款仍以订单级人工记录为主，商品级退款、退货入库、换货和补差价留给后续版本。
- PWA 线上更新后，旧缓存刷新体验仍需进一步优化，建议进入 `v1.5.4` 独立处理。
