# ECRM——Easy CRM

为离线而生（线下摊主使用）的轻量CRM进销存管理系统，当前适用于个人原创/同人衍生周边的现场商品管理、开单、促销计算、手动收款确认、库存扣减、备份和极简仪表盘。

## 当前功能范围

已实现：

- 商品管理：新增、编辑、停用、图片上传、售价/成本/库存、可售卖/可赠品标记、排序。
- 售卖管理：SPU 筛选、商品加购、购物车数量调整、加购优惠、满赠提示、库存保护、暂存购物车、收款码展示、手动确认已收款、订单保存和库存扣减。
- 促销规则：任意消费后指定 SPU 前 3 个优惠；仅买优惠 SPU 时第 1 个原价、第 2-4 个优惠、第 5 个起原价；满赠按最高档生效。
- 订单售后：订单详情、订单作废、退款记录、库存回滚和库存流水。
- 设置：店铺名、订单前缀、微信/支付宝收款码、优惠 SPU、赠品 A/B、JSON 数据备份和恢复、订单 Excel 导出。
- 仪表盘：日期范围切换、销售额、订单数、客单价、利润、SKU/SPU 排名、促销/收款统计、库存风险。
- 现场模式：设置 4 位数字 PIN 后，商品、设置、仪表盘进入前需要解锁；售卖页保持可用。
- PWA：可构建为静态站点，部署到 HTTPS 后可在 iPad Safari 添加到主屏幕并离线使用。

暂不实现：

- 复合支付二维码。
- 微信/支付宝支付 API 或自动查账。
- 云端同步、多设备合并。
- 飞书多维表格式自定义仪表盘。
- 小票打印、扫码枪等外设。

## 本地开发

```powershell
npm install
npm run dev
```

默认开发地址：

```text
http://localhost:5173/
```

局域网设备访问时，需要使用运行电脑的局域网 IP，例如：

```text
http://192.168.x.x:5173/
```

注意：iPad “添加到主屏幕并离线安装”需要 HTTPS。局域网 HTTP 只能用于临时预览和调试，不能作为最终 iPad PWA 安装方案。

## 免费 HTTPS 部署

没有 Apple 开发者账号、签名、App Store、TestFlight、自有服务器或域名时，推荐使用免费静态托管：

- Vercel
- Netlify
- Cloudflare Pages
- GitHub Pages

最快方式：

1. 本地运行 `npm run build`。
2. 将 `dist/` 上传到 Vercel/Netlify/Cloudflare Pages 的静态站点部署入口，或用 GitHub Pages 发布。
3. 获得一个 HTTPS 地址。
4. 在 Windows/Mac/iPad 浏览器打开该 HTTPS 地址。

## iPad 安装与离线验收

首次安装必须联网打开 HTTPS 地址。

验收步骤：

1. 在 iPad Safari 打开部署后的 HTTPS 地址。
2. 点击分享按钮，选择“添加到主屏幕”。
3. 从主屏幕图标打开 ECRM。
4. 录入商品 A：售价 5，允许售卖，允许赠品。
5. 录入商品 B：售价 0，不允许售卖，允许赠品。
6. 录入普通商品：售价 20，允许售卖。
7. 在设置中配置优惠 SPU 为商品 A 所属 SPU，优惠价 3，最多优惠 3 个。
8. 在设置中选择 A 赠品和 B 赠品。
9. 上传微信和支付宝收款码。
10. 在售卖页只添加商品 A x1，确认应收为 `¥5.00`。
11. 只添加商品 A x4，确认应收为 `¥14.00`。
12. 添加普通商品 x1 和商品 A x4，确认应收为 `¥34.00`。
13. 添加商品达到 `¥68.00` 档，确认购物车显示 A x2 + B x1 的赠品信息。
14. 确认收款并保存订单。
15. 回到商品页确认普通商品、商品 A、赠品库存已扣减。
16. 打开仪表盘，确认今日销售额和订单数更新。
17. 关闭应用，打开飞行模式。
18. 从主屏幕重新打开应用。
19. 再创建一笔订单，确认订单和库存仍能离线保存。

## 摆摊前检查

- 商品名称、SPU、售价、库存已录入。
- 商品图已上传，并确认列表内显示比例正常。
- 商品 A 已设置为可售卖、可赠品。
- 商品 B 已设置为不可售卖、可赠品。
- 微信和支付宝收款码已上传并保存设置。
- 加购优惠 SPU、优惠价、最多优惠数量已保存。
- 赠品 A/B 已选择。
- 用测试订单确认优惠金额、满赠档位、库存扣减正确。
- 导出一份 JSON 备份。
- 在 iPad 飞行模式下完整走一次开单和保存。

## 备份说明

设置页支持导出/导入 JSON 备份。JSON 备份包含：

- 商品资料。
- 设置和促销配置。
- 订单。
- 订单明细。
- 库存流水。
- 退款记录。
- 商品图和收款码图片。

现场模式 PIN 不会随 JSON 备份导出。导入备份到新设备后，现场模式默认关闭，需要在目标设备重新设置 PIN。

## 数据与离线说明

- 数据保存在当前浏览器/设备的 IndexedDB 中。
- 同一 HTTPS 站点在同一浏览器内可持续离线使用。
- 清除浏览器网站数据、卸载 PWA、换浏览器、换设备，都会导致本地数据不可见。
- 当前版本没有云同步，多设备不会自动合并数据。
- 确认已收款后才会保存订单并扣减库存。
- 库存不足会阻止继续加购或阻止保存订单。
- 现场模式用于现场防误触和隐私遮挡，不是强加密或多用户权限系统。
